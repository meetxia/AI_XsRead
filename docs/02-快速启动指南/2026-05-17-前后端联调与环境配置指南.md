# 前后端联调与环境配置指南

文档日期：2026-05-17

本文聚焦本地联调：端口、代理、环境变量、请求路径和排错口径。所有用户端 API 走用户后端 `8005`，所有管理端 API 走管理后端 `8001`。

## 联调拓扑

```text
用户浏览器
  -> http://localhost:3008
  -> Vite proxy /api      -> http://localhost:8005/api
  -> Vite proxy /uploads  -> http://localhost:8005/uploads

管理浏览器
  -> http://localhost:3010
  -> Vite proxy /api      -> http://localhost:8001/api
```

## 用户端配置

相关文件：

- `ai-xsread-vue3/vite.config.js`
- `ai-xsread-vue3/.env`
- `ai-xsread-vue3/.env.development`
- `ai-xsread-vue3/src/api/request.js`

当前口径：

```js
server: {
  port: 3008,
  host: '0.0.0.0',
  proxy: {
    '/api': {
      target: 'http://localhost:8005',
      changeOrigin: true
    },
    '/uploads': {
      target: 'http://localhost:8005',
      changeOrigin: true
    }
  }
}
```

前端请求基址：

```env
VITE_APP_BASE_API=/api
```

用户端 Axios 会读取 `import.meta.env.VITE_APP_BASE_API`。开发环境保持 `/api`，让 Vite 代理处理跨端口；生产环境也保持 `/api`，由 Nginx 或部署层反向代理到用户后端。

## 用户后端配置

相关文件：

- `backend/src/app.js`
- `backend/src/config/index.js`
- `backend/src/routes/index.js`
- `backend/.env`

最小可用配置：

```env
PORT=8005
NODE_ENV=development
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=your_database_user
DB_PASSWORD=your_database_password
DB_DATABASE=ai_xsread
JWT_SECRET=your_jwt_secret_key_at_least_32_characters_long
JWT_REFRESH_SECRET=your_refresh_secret_key_at_least_32_characters_long
JWT_EXPIRES_IN=7d
JWT_REFRESH_EXPIRES_IN=30d
CORS_ORIGIN=http://localhost:3008
UPLOAD_DIR=uploads
```

用户后端当前挂载：

- `/api/health`：健康检查
- `/api/auth/*`：注册、登录、刷新、退出
- `/api/novels*`、`/api/chapters/*`：小说和章节
- `/api/user/*`：书架、进度、资料、成就等登录态接口
- `/api/upload/*`：TXT、头像、图片上传
- `/uploads/*`：本地上传静态资源

## 管理端配置

相关文件：

- `admin-frontend/vite.config.js`
- `admin-frontend/.env`
- `admin-frontend/src/utils/request.js`

当前口径：

```js
server: {
  port: 3010,
  proxy: {
    '/api': {
      target: 'http://localhost:8001',
      changeOrigin: true
    }
  }
}
```

前端请求基址：

```env
VITE_APP_BASE_API=/api
```

管理前端 Axios 默认从 `localStorage.admin_token` 读取后台登录令牌，并把它放入 `Authorization: Bearer ...`。

## 管理后端配置

相关文件：

- `admin-backend/src/app.js`
- `admin-backend/src/config/index.js`
- `admin-backend/.env`

最小可用配置：

```env
PORT=8001
NODE_ENV=development
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=your_database_user
DB_PASSWORD=your_database_password
DB_NAME=ai_xsread
JWT_SECRET=your_jwt_secret_key_at_least_32_characters_long
JWT_EXPIRES_IN=2h
JWT_REFRESH_EXPIRES_IN=7d
CORS_ORIGIN=http://localhost:3010
UPLOAD_DIR=./uploads
```

注意：管理后端数据库变量名是 `DB_NAME`；用户后端是 `DB_DATABASE`。二者连接同一个库 `ai_xsread`。

2. 启动用户后端：

## 后台管理系统完整启动核对

管理后台至少需要完成以下事项：

1. `admin-backend/.env` 使用 `DB_NAME=ai_xsread`，并配置不少于 32 字符的 `JWT_SECRET`。
2. 数据库中已存在用户端基础表。新库先导入 `backend/database/init_step1.sql`。
3. 执行管理端表检查或创建：

```powershell
cd E:\momo-ruanjiansheji\AI-XsRead\admin-backend
node scripts/check-and-create-tables.js
node scripts/check-all-tables.js
```

4. 初始化管理员账号：

```powershell
node scripts/init-admin.js
```

5. 启动 `admin-backend` 后访问 `http://localhost:8001/api/health`。
6. 启动 `admin-frontend` 后访问 `http://localhost:3010/login`。

## 排错速查

| 现象 | 优先检查 |
| --- | --- |
| 用户前端请求打到旧端口 | `ai-xsread-vue3/.env*` 是否仍为 `VITE_APP_BASE_API=/api` |
| 用户端 `/uploads/...` 404 | 用户后端是否运行在 `8005`，资源是否在 `backend/uploads/` |
| 管理端登录接口失败 | 管理后端是否运行在 `8001`，管理前端代理是否走 `/api` |
| 后端报数据库环境变量缺失 | 用户后端用 `DB_DATABASE`，管理后端用 `DB_NAME` |
| CORS 报错 | 用户后端 `CORS_ORIGIN=http://localhost:3008`；管理后端 `CORS_ORIGIN=http://localhost:3010` |
| 401 后被跳转登录 | 检查浏览器 localStorage 中的 `token` 或 `admin_token` 是否过期 |

## 不再推荐的旧口径

以下旧口径来自历史文档或模板，当前不要继续使用：

- 用户后端默认 `8000`
- 用户前端默认 `3000`、`5173` 或 `8080`
- 管理前端默认 `8080`
- 管理后端连接变量写成 `DB_DATABASE`
- 开发环境把前端 API 写成硬编码完整 URL
