# 本地快速启动总览

文档日期：2026-05-17

本文是 AI-XsRead（文字之境）本地开发的第一入口，统一用户端、管理端、后端服务的当前启动口径。旧文档中的 `8000`、`3000`、`8080` 等端口不再作为当前项目默认端口使用。

## 当前服务矩阵

| 系统 | 目录 | 技术栈 | 默认端口 | 主要命令 | 代理关系 |
| --- | --- | --- | --- | --- | --- |
| 用户前端 | `ai-xsread-vue3/` | Vue 3 + Vite + TailwindCSS + Pinia | `3008` | `npm run dev` | `/api` 和 `/uploads` 代理到 `http://localhost:8005` |
| 用户后端 | `backend/` | Express 4 + MySQL2 + JWT + Redis 可选 | `8005` | `npm run dev` | 提供用户端 API、上传静态资源 |
| 管理前端 | `admin-frontend/` | Vue 3 + Vite + Element Plus + ECharts | `3010` | `npm run dev` | `/api` 代理到 `http://localhost:8001` |
| 管理后端 | `admin-backend/` | Express 5 + MySQL2 + JWT | `8001` | `npm run dev` | 提供后台管理 API |

## 推荐启动顺序

先启动数据库，再启动两个后端，最后启动两个前端。

```powershell
# 1. 用户后端：localhost:8005
cd E:\momo-ruanjiansheji\AI-XsRead\backend
npm install
npm run dev
```

```powershell
# 2. 管理后端：localhost:8001

npm install
npm run dev
```

```powershell

cd E:\momo-ruanjiansheji\AI-XsRead\ai-xsread-vue3
npm install
npm run dev
```

```powershell
# 4. 管理前端：localhost:3010

```

## 环境变量最小配置

用户后端使用 `DB_DATABASE`，管理后端使用 `DB_NAME`。二者当前都应指向同一个业务库：`ai_xsread`。

`backend/.env` 至少需要：

```env
PORT=8005
NODE_ENV=development
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=your_database_user
DB_PASSWORD=your_database_password
DB_DATABASE=ai_xsread
JWT_SECRET=your_jwt_secret_key_at_least_32_characters_long
JWT_REFRESH_SECRET=your_refresh_secret_key_at_least_32_characters_long
CORS_ORIGIN=http://localhost:3008
UPLOAD_DIR=uploads
```

`admin-backend/.env` 至少需要：

```env
PORT=8001
NODE_ENV=development
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=your_database_user
DB_PASSWORD=your_database_password
DB_NAME=ai_xsread
JWT_SECRET=your_jwt_secret_key_at_least_32_characters_long
CORS_ORIGIN=http://localhost:3010
UPLOAD_DIR=./uploads
```

前端默认走同源相对路径，不需要把 API 写死成完整 URL：

```env
VITE_APP_BASE_API=/api
```

## 一次性初始化事项

1. 创建 MySQL 数据库 `ai_xsread`，字符集建议 `utf8mb4`。
2. 执行用户后端基础建表脚本：`backend/database/init_step1.sql`。
3. 如需要后台账号表，进入 `admin-backend/` 后执行：

```powershell
node scripts/check-and-create-tables.js
node scripts/init-admin.js
```

管理员账号以 `init-admin.js` 的实际输出为准。本地调试可以使用脚本生成的默认账号，生产和演示环境必须立即改密码。

4. 如需要导入 `data/` 下的 TXT 小说，进入 `backend/` 后执行：

```powershell
node scripts/import-txt-novels.js
```

## 常见问题

### 前端请求 404 或 Network Error

优先确认前端是否从 Vite 开发服务器访问：

- 用户端：`http://localhost:3008`
- 管理端：`http://localhost:3010/login`

开发环境不要直接访问后端 HTML，也不要把 `VITE_APP_BASE_API` 改成旧端口。Vite 会根据 `vite.config.js` 代理 `/api`。

### 用户端图片或头像不显示

用户前端开发环境已配置 `/uploads` 代理到 `http://localhost:8005`。请确认用户后端已启动，并且上传资源在 `backend/uploads/` 下。

### 管理后端启动报缺少环境变量

管理后端要求 `DB_HOST`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`、`JWT_SECRET`。注意这里是 `DB_NAME`，不是用户后端的 `DB_DATABASE`。

### 用户后端启动报 JWT 密钥过短

`JWT_SECRET` 和 `JWT_REFRESH_SECRET` 都需要至少 32 个字符。可以用 Node 生成：

```powershell
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```
