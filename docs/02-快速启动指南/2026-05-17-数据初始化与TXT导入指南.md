## 初始化数据库

先创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS ai_xsread
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

导入基础表结构：

```powershell
mysql -u your_database_user -p ai_xsread < E:\momo-ruanjiansheji\AI-XsRead\backend\database\init_step1.sql
```

如果使用 phpMyAdmin：

1. 选择或新建数据库 `ai_xsread`。
2. 进入“导入”。
3. 选择 `backend/database/init_step1.sql`。
4. 字符集选择 `utf8mb4`。
5. 执行导入后确认 `users`、`novels`、`chapters`、`categories` 等基础表存在。

## 批量导入 data 目录 TXT 小说

仓库根目录的 `data/` 保存 TXT 小说文件。当前批量导入脚本会扫描该目录下的 `.txt` 文件，并写入 `novels` 和 `chapters`。

执行方式：

```powershell
cd E:\momo-ruanjiansheji\AI-XsRead\backend
node scripts/import-txt-novels.js
```

脚本行为：

- 从 `backend/.env` 读取数据库配置。
- 使用 `DB_DATABASE` 连接 `ai_xsread`。
- 以 TXT 文件名作为小说标题。
- 将全文作为单章“正文”导入。
- 根据标题关键词映射分类。
- 如果同名小说已存在，会跳过，不重复插入。

导入完成后可访问：

```text
http://localhost:3008
```

## 用户端上传 TXT 小说

用户端提供登录后的 TXT 上传入口：

```text
http://localhost:3008/upload
```

当前接口：

| 功能 | 方法与路径 | 表单字段 | 鉴权 |
| --- | --- | --- | --- |
| 上传单本 TXT | `POST /api/upload/novel` | `file` | 用户登录 |
| 批量上传 TXT | `POST /api/upload/novels/batch` | `files` | 用户登录 |
| 获取我的上传小说 | `GET /api/upload/my-novels` | - | 用户登录 |

开发环境中，用户前端 `http://localhost:3008` 会把 `/api/upload/*` 代理到 `http://localhost:8005/api/upload/*`。

上传限制来自后端 Multer 配置：

- 仅允许 `.txt` 文件。
- 单次批量接口最多 50 个文件。
- 单个 TXT 上传接口限制 10MB。
- 文件会先落到 `backend/uploads/novels/`，再解析入库。

后端解析规则：

- 文件名去掉 `.txt` 后作为小说标题。
- 正文作为单章“正文”导入。
- 自动去除 BOM 并统一换行符。
- 简介取正文前 200 字，优先选取较长内容段。
- 标题关键词会映射到现代言情、古风穿越、悬疑推理、治愈系、玄幻奇幻、科幻未来等分类。
- 同名小说会被判定为已存在，避免重复导入。

PowerShell 单本上传示例：

```powershell
$token = "替换为登录后的用户token"
$file = "E:\momo-ruanjiansheji\AI-XsRead\data\不会说话的你，被政界新贵指定联姻.txt"

curl.exe -X POST "http://localhost:8005/api/upload/novel" `
  -H "Authorization: Bearer $token" `
  -F "file=@$file"
```

PowerShell 批量上传示例：

```powershell
$token = "替换为登录后的用户token"

curl.exe -X POST "http://localhost:8005/api/upload/novels/batch" `
  -H "Authorization: Bearer $token" `
  -F "files=@E:\momo-ruanjiansheji\AI-XsRead\data\不会说话的你，被政界新贵指定联姻.txt" `
  -F "files=@E:\momo-ruanjiansheji\AI-XsRead\data\大师劝复合，我闺蜜差点信了爱情.txt"
```

## TXT 上传验收清单

- 上传接口返回统一格式 `{ code, message, data, timestamp }`。
- `data.novelId` 存在。
- 用户端小说详情页可打开。
- 阅读页能显示“正文”章节。
- 管理端小说列表和章节列表可看到对应记录。

## 种子数据脚本说明

`backend/scripts/import-seed-data.js` 是旧的种子导入脚本，仍保留在代码中，但它读取 SQL 文件路径的历史口径可能与当前迁移后的文档目录不完全一致。新环境优先使用：

1. `backend/database/init_step1.sql` 初始化基础结构。
2. `admin-backend/scripts/check-and-create-tables.js` 创建或检查后台表。
3. `admin-backend/scripts/init-admin.js` 创建后台管理员。
4. `backend/scripts/import-txt-novels.js` 导入 `data/` 下的小说内容。

如确实需要导入完整历史种子 SQL，请先确认 SQL 文件实际所在位置，再执行手动导入，避免把迁移中的旧 `docx/` 路径当作稳定入口。

## 常见问题

### `ER_BAD_DB_ERROR`

数据库 `ai_xsread` 不存在。先创建库，再导入 `backend/database/init_step1.sql`。

### `ER_ACCESS_DENIED_ERROR`

数据库用户名或密码错误。检查 `backend/.env` 和 `admin-backend/.env`。

### 批量导入提示同名已存在

这是脚本的防重复逻辑，不是失败。需要重导时应先确认是否要清理对应小说和章节数据。

### 用户端上传后看不到小说

确认当前登录用户、后端 `8005`、数据库写入结果，以及前端是否通过 `http://localhost:3008` 访问。
