## 用户端 UI 口径

用户端目录：

```text
ai-xsread-vue3/
  src/
    views/
    components/
      common/
      bookshelf/
      novel/
      reading/
      profile/
      search/
      v2/
    composables/
    stores/
    api/
    router/
```

技术与风格：

- Vue 3 `<script setup>`。
- TailwindCSS 3.4。
- Pinia 管理书架、主题、用户状态等。
- 移动端优先，兼顾桌面端阅读。
- 当前主题重点包括玫瑰、薰衣草、护眼和夜间阅读体验。

| 页面 | 说明 |
| --- | --- |
| 首页 | 推荐、分类、继续阅读入口 |
| 搜索页 | 搜索建议、热门搜索、结果列表 |

## 用户端组件与组合式函数

常用组件目录：

- `components/common/`：通用头部、导航、基础展示组件。
- `components/novel/`：小说卡片、章节列表、评论、评分、推荐栏等。
- `components/reading/`：阅读器相关抽屉、控制面板、阅读辅助组件。
- `components/bookshelf/`：书架分组、批量操作、未读标记。
- `components/profile/`：个人中心、设置、安全、统计展示。
- `components/search/`：增强搜索栏等。
- `components/v2/`：新版书卡、封面、图标、布局与主题基础组件。

## 阅读体验当前要求

阅读页是用户端核心体验，调整时需要重点检查：

- 字号、行高、段落间距切换后正文不溢出。
- 夜间、护眼、默认主题下文字对比度可读。
- 目录抽屉、设置面板、段评入口在移动端不遮挡正文核心操作。
- 阅读进度在章节切换、退出页面、重新进入时表现一致。
- 登录过期时应回到登录流程；匿名访问可读内容不应被无意义强跳转打断。

## 微交互保留原则

旧文档中有大量模板化动画示例，当前不再作为实现规范。保留以下原则：

- 交互动效以确认状态变化为主，例如按钮按下、加载、保存成功、面板展开。
- 阅读器内动效要克制，不能影响长时间阅读。
- 列表、卡片、抽屉、弹层的动画时长保持短而一致。
- 不为了装饰增加额外状态和复杂依赖。

可复用微交互组件：

- `common/LikeButton.vue`：点赞。
- `common/RippleEffect.vue`：点击反馈。
- `common/PullToRefresh.vue`：移动端下拉刷新。
- `common/BackToTop.vue`：返回顶部。
- `reading/PageFlipController.vue`：翻页控制。
- `reading/HighlightToolbar.vue`：文本选择后操作。
- `reading/ParagraphCommentBubble.vue`：段落评论提示。

新增微交互至少检查默认态、触摸按下态、加载态、失败态、深色主题和小屏幕表现。

## 管理端 UI 口径

管理端目录：

```text
admin-frontend/
  src/
    views/
      Dashboard/
      Content/
      Users/
      Comments/
      Analytics/
      Settings/
      Login/
    components/
    router/
    store/
    utils/
```

技术与风格：

- Vue 3 `<script setup>`。
- Element Plus 2.11。
- ECharts 6。
- SCSS 变量。
- 以后台效率为主，优先清晰的信息密度、稳定表格、明确筛选和操作反馈。

管理端新增功能时，优先使用 Element Plus 已有组件：

- 表格：`el-table`
- 表单：`el-form`
- 弹窗：`el-dialog`
- 分页：`el-pagination`
- 上传：`el-upload`
- 图表：ECharts

## 设计与实现边界

| 场景 | 推荐做法 |
| --- | --- |
| 用户端读者体验 | TailwindCSS + 现有主题变量 + 已有 reading/profile/novel 组件 |
| 管理后台表格和表单 | Element Plus + 管理端 API 封装 |
| 上传、登录、权限 | 走现有接口封装，不在组件里硬编码 URL |
| 图片和头像展示 | 使用同源 `/uploads/...` 路径，开发环境由 Vite 代理 |
| 新增通用逻辑 | 抽到 `composables/` 或 `utils/` |

## UI 联调检查清单

用户端：

- `http://localhost:3008` 首页可加载。
- 搜索、详情、阅读、书架、个人中心主要路径可进入。
- 阅读设置面板在桌面和移动端均可操作。
- 登录态接口不会让匿名浏览页频繁跳转。
- 上传页只接受 `.txt`，上传进度有反馈。

管理端：

- `http://localhost:3010` 可打开登录页。
- 登录后 Dashboard 可加载概览数据。
- 内容、章节、用户、评论页面的表格分页正常。
- 表单提交失败时有明确错误提示。
