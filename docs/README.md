## 5. 口径冲突时怎么办

当文档之间冲突时，按以下顺序判断：

1. 当前代码实现
2. `docs/项目总文档/`
3. `docs/01-核心文档/`
4. `docs/02-快速启动指南/`
5. 最新日期的 `docs/AI完成后报告文档/`
6. `docs/历史研究归档/`
7. `docx/` 或迁移前备份

一句话：当前事实看总文档和代码，历史判断看归档，旧目录不要再当主入口。
