## 5. 口径冲突时怎么办

按以下顺序判断：

1. 当前代码实现
2. `docs/项目总文档/2026-05-21-*.md`（A 级最新）
3. `docs/01-核心文档/2026-05-21-*.md`（A 级最新）
4. `docs/02-快速启动指南/`
5. 最新日期的 `docs/AI完成后报告文档/`（B 级，但**单次任务范围内**权威）
6. `docs/项目总文档/2026-05-17-*.md` / `docs/01-核心文档/2026-05-17-*.md`（5/17 版本，已被 5/21 取代）
7. `docs/历史研究归档/`
8. `docx/` 或迁移前备份

一句话：**当前事实看 5/21 总文档和代码，历史判断看归档，5/17 旧版只作演进对照。**

## 6. 5/17 → 5/21 的演进对照

5/21 版本相比 5/17 版本主要补：

- 个人中心 MVP（编辑资料、阅读偏好、通知中心）
- 会员体系与激活码（用户端会员入口 + 管理端激活码批次/单码/兑换日志/操作审计）
- 管理端 Settings / Analytics / 系统模块全链路
- 数据库迁移：从 19 个迁移文件涨到 25 个，新增 user_preferences / notifications / admin_users / admin_logs / banners / announcements / statistics_daily / system_contact / 会员字段 / 激活码 4 表
- 工程化：CI 全闸门必绿、Jest/Vitest 接入 4 子工程、deploy.py 唯一部署、Nginx HTTPS + HSTS、迁移命名规范
- 安全：admin/admin123 默认密码废除、JWT 双密钥 + HS256 锁定、SQL 注入白名单、上传 MIME 严格白名单、登录注册专项限流

5/17 旧版文档保留作历史对照，**不要再当当前事实读**。
