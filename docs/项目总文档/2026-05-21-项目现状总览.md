# MOMO小说（AI-XsRead）项目现状总览

文档日期：2026-05-21（取代 [2026-05-17 版](2026-05-17-项目现状总览.md)）

## 1. 项目定位

MOMO小说（AI-XsRead）是一个面向女性用户的小说阅读平台，前后端分离，由 4 个核心子工程组成：

1. 用户端前端：`ai-xsread-vue3`（端口 3008）
2. 用户端后端：`backend`（端口 8005）
3. 管理端前端：`admin-frontend`（端口 3010）
4. 管理端后端：`admin-backend`（端口 8001）

**当前阶段**：用户端 Sprint 1（2026-05-20）+ 管理端 Sprint 2（2026-05-21）双 Sprint 完成，用户端主链路、个人中心 MVP、会员激活、管理端 8 大模块、SEO、CI、部署脚本全部上线，**项目具备整站对外发布能力**，仅待 staging 真实环境演练做最后验收。

## 2. 当前技术栈

### 用户端前端 `ai-xsread-vue3`
- Vue 3.5 / Vite 7 / Vue Router 4.6 / Pinia 3.0
- TailwindCSS 3.4
- Axios（含 DEV 日志脱敏）
- Vitest + Playwright + Vue Test Utils（23 套件 / 107 通过）

### 用户端后端 `backend`
- Express 4.18 / MySQL2 / JWT（HS256 + 双密钥）/ Redis
- Helmet、CORS、Rate Limit（含登录注册专项 10/15min）、Multer、Swagger（仅 dev 暴露）
- node-cron（定时任务）
- Jest + Supertest（42 套件 / 251 通过 / 2 skip）
- 自动迁移流水线 `database/migrations/migrate.js`

### 管理端前端 `admin-frontend`
- Vue 3.5 / Vite 7
- Element Plus 2.11 / ECharts 6 / SCSS
- Vitest（5 条静态守护）

### 管理端后端 `admin-backend`
- Express 5.1 / MySQL2 / JWT（HS256 + 双密钥）
- Multer + Express Validator + Compression
- 上传 MIME 严格白名单（jpeg/jpg/png/webp，**拒绝 SVG**）
- 启动期 `db.query('SELECT 1')` 探活 + SIGINT 优雅关闭 + `trust proxy: 1`
- Jest + Supertest（10 条 smoke）

### 数据库
- MySQL 8.0
- 字符集 `utf8mb4`，排序 `utf8mb4_unicode_ci`
- 业务库 `ai_xsread`，由用户后端与管理后端**共享**
- 25 个迁移文件管理 30+ 张业务表（详见 [数据库与迁移现状说明](2026-05-21-数据库与迁移现状说明.md)）

### 工程化与基础设施
- GitHub Actions CI（4 子工程 build + test 全闸门必绿，无 `continue-on-error`）
- 部署：`backend/scripts/deploy.py` 唯一权威（rsync `--checksum` 增量 + PM2 reload + 自动迁移诊断）
- Nginx：用户域 + admin 子域两个 server block + HSTS / X-Frame-Options / Permissions-Policy / TLS 1.2/1.3
- PM2：`xsread-backend`（8005）+ `xsread-admin-backend`（8001）
- 静态资源：双格式（JPG + WebP）+ Gzip + Brotli

## 3. 四端运行边界

### 用户端前端
- 开发端口：`3008`
- 代理：`/api`、`/uploads` → `http://localhost:8005`

### 用户端后端
- 服务端口：`8005`
- 健康检查：`/api/health`
- 公开端点：`/api/health` / `/` / `/api/system/*` / SEO（`/sitemap*.xml` / `/robots.txt`）
- 全局限流：dev 3000 次/15min，production 默认值（跳过 `/api/health` 与 `/uploads/`）

### 管理端前端
- 开发端口：`3010`
- 代理：`/api` → `http://localhost:8001`

### 管理端后端
- 服务端口：`8001`
- 健康检查：`/health`
- 全部业务路径以 `/admin` 开头（`/admin/login` / `/admin/dashboard` / `/admin/novels` / `/admin/codes` / `/admin/system` 等）

## 6. 安全姿态（双 Sprint 后）

| 项 | 改前 | 改后 |
|---|---|---|
| admin 默认密码 | `admin/admin123` 硬编码 | 20 位随机密码写入 `.admin-credentials`（mode 0o600） |
| JWT 算法 | 默认（隐式 HS256） | 显式锁 HS256，杜绝 `algorithm: none` 攻击 |
| JWT 密钥 | 单密钥 access+refresh 共用 | access 与 refresh 拆分双密钥，强校验 ≥ 32 位 |
| 后端 token 日志 | 每请求打印前 20 字符 + userId/username | 完全无日志 |
| 前端 token 日志 | DEV 打印 tokenPreview + authHeader 前 30 字符 | DEV 仅打印 method + url + (authed/anon) |
| `/api-docs` swagger | 生产环境裸暴露 | 生产环境 404 |
| 全局限流 | 仅 production 启用 | 全环境启用（dev 3000/15min） |
| 登录/注册限流 | 仅全局桶 | 专项 10/15min 桶 |
| SQL 注入（管理端 novelController.getList） | `ORDER BY ${sort}` 用户可控 | 白名单 `ALLOWED_SORTS` + `sortField` 派生 |
| 上传 MIME | 宽松匹配，可传 SVG | 严格白名单 jpeg/jpg/png/webp，拒绝 SVG（防内嵌 JS XSS） |
| 错误堆栈泄漏 | errorHandler 打印完整 err 含 stack/sqlMessage/req.body | 字典化错误码 + 仅打印安全字段 + 生产 5xx 文案统一 |
| `trust proxy` | 未设置（限流误判 IP） | `app.set('trust proxy', 1)` |
| Nginx 安全头 | 旧 CORS `*` + 无 HSTS | HSTS / X-Frame-Options DENY / X-Content-Type / Referrer-Policy / Permissions-Policy + HTTP→HTTPS 301 |
| csurf 死代码 | 占位但未挂载 | 删除 + 移除 12 个传递依赖 |
| CI 测试闸门 | `continue-on-error: true` 假绿 | 失败必红 |
| 外链 referrer | NotificationsPage 仅 noopener | noopener + noreferrer |
