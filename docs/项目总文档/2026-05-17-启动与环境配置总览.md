# 文字之境（AI-XsRead）启动与环境配置总览

文档日期：2026-05-17

## 1. 总体说明

本项目包含用户端前端、用户端后端、管理端前端、管理端后端四个独立应用。开发时通常需要按“前端 + 后端”成组启动，用户端与管理端互不干扰。

## 2. 启动命令

### 用户端前端
目录：`ai-xsread-vue3`

```bash
npm install
npm run dev
npm run build
npm run preview
npm run test
npm run test:watch
```

### 用户端后端
目录：`backend`

```bash
npm install
npm run dev
npm start
npm run test
npm run test:watch
npm run test:unit
npm run test:integration
```

### 管理端前端
目录：`admin-frontend`

```bash
npm install
npm run dev
npm run build
npm run preview
```

### 管理端后端
目录：`admin-backend`

```bash
npm install
npm run dev
npm start
```

## 3. 端口说明

- 用户端前端：`3008`
- 用户端后端：`8005`
- 管理端前端：`3010`
- 管理端后端：`8001`

## 4. 关键环境变量

### 用户端后端
来自 `backend/src/config/index.js` 的实际校验逻辑，至少需要：

- `DB_HOST`
- `DB_USER`
- `DB_PASSWORD`
- `DB_DATABASE`
- `JWT_SECRET`
- `JWT_REFRESH_SECRET`

常见可选项还包括：

- `PORT`
- `DB_PORT`
- `DB_CONNECTION_LIMIT`
- `DB_QUEUE_LIMIT`
- `DB_ACQUIRE_TIMEOUT`
- `DB_TIMEOUT`
- `DB_CONNECT_TIMEOUT`
- `JWT_EXPIRES_IN`
- `JWT_REFRESH_EXPIRES_IN`
- `CORS_ORIGIN`
- `UPLOAD_DIR`
- `MAX_FILE_SIZE`

### 管理端后端
管理端后端采用独立环境变量配置，重点包含：

- `PORT`
- `NODE_ENV`
- `DB_HOST`
- `DB_PORT`
- `DB_USER`
- `DB_PASSWORD`
- `DB_NAME`
- `JWT_SECRET`
- `JWT_EXPIRES_IN`
- `JWT_REFRESH_EXPIRES_IN`
- `UPLOAD_DIR`
- `MAX_FILE_SIZE`
- `CORS_ORIGIN`

## 5. 本地联调建议

1. 先启动数据库并导入基础表结构与种子数据。
2. 先启动后端，再启动前端。
3. 用户端开发时，前端代理会自动把 `/api` 和 `/uploads` 指向 `8005`。
4. 管理端开发时，前端代理会把 `/api` 指向 `8001`。
5. 如果出现图片、封面、头像不显示，优先排查后端是否启动以及上传目录是否可访问。

## 6. 常见误区

1. 不要混用用户端与管理端端口。
2. 不要沿用旧文档里的模板端口或旧数据库名。
3. 不要把 `docx/` 中的老旧快速开始说明直接当成当前真相。
4. 任何“登录成功但页面仍然跳回登录页”的问题，优先检查 token 读取、路由守卫、以及 API baseURL 是否正确。
