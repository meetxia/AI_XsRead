# 中长期产品路线图

文档日期：2026-05-17

## 1. 说明

本文重写 2025-10-27 旧“V1.1 / V1.5 / V2.0”路线图。旧版本号和人天预估不再作为当前承诺。

当前路线图改用阶段制，避免把尚未验证的商业化、作者生态、多端和 AI 功能过早承诺为固定版本。

## 2. 阶段 A：上线可信度收敛

目标：让现有能力可验收、可部署、可维护。

重点：
- 文档口径统一
- 真实验收矩阵
- 数据真实性
- 数据库迁移边界
- 部署检查清单
- 核心 e2e 覆盖

完成标志：
- 主链路可重复验收
- 旧文档已归档
- README 和快速启动指南可信
- 线上深链、API、上传资源可用

## 3. 阶段 B：阅读体验差异化

目标：让阅读体验从“可用”变成“愿意长期使用”。

重点：
- 阅读页多断点体验
- TTS 与翻页体验
- 段评、划线、书签、想法一体化
- 阅读设置稳定
- 书架和历史真实同步
- 成就和连读反馈

完成标志：
- 重度读者主流程顺畅
- 移动端阅读体验稳定
- 阅读行为数据可靠

## 4. 阶段 C：内容运营与社区增强

目标：提升找书效率、内容质量和互动留存。

重点：
- 内容导入和清洗
- 分类、标签、封面、章节质量
- 推荐解释
- 热门榜单
- 评论和段评治理
- 关注作者和更新提醒

完成标志：
- 内容质量可控
- 用户能快速找到想读的书
- 社区互动有治理能力

## 5. 阶段 D：商业化与作者生态

目标：在内容规模、留存和运营后台成熟后探索商业化。

可能方向：
- 作者后台
- VIP
- 付费章节
- 打赏
- 版权保护
- 数据看板升级

前置条件：
- 内容供给稳定
- 用户行为数据可信
- 管理后台能支撑内容运营
- 支付和权限体系经过安全评估

## 6. 阶段 E：多端与 AI 探索

目标：在 Web 主链路稳定后探索多端和 AI 增强。

可能方向：
- PWA
- 小程序
- App 壳
- AI 摘要
- AI 推荐解释
- AI 辅助书评
- 智能客服

原则：
- 不为了“AI”牺牲阅读主体验
- 不在数据基础薄弱时做高成本推荐系统
- 多端建设必须复用稳定的 Web 能力
