# AI-XsRead 下一阶段工作任务书

文档日期：2026-05-19

## 1. 任务书定位

本文档用于指导 AI-XsRead 接下来一轮开发收口与排期执行。它不是愿望清单，而是基于当前实际代码、最新提交与现状核查报告形成的负责人任务书。

参考依据：

- 最新提交：`4048760 feat(profile): 阶段性提交个人中心MVP进展`
- 现状报告：`docs/AI完成后报告文档/2026-05-19-AI-XsRead项目现状核查报告.md`
- 当前状态：`main` 分支工作区干净；是否推送以执行时 `git status --short --branch` 为准

本阶段总目标：

1. 恢复四端质量闸门，先让构建和测试结果可信。
2. 收口个人中心 MVP，使其从“阶段性提交”进入“可验收功能”。
3. 补齐管理端最小自动化测试，避免后台继续裸奔。
4. 为后续功能迭代清理高风险技术债入口。

## 3. 工作原则

1. 先修质量闸门，再继续加功能。
2. 所有修复必须以实际命令输出作为完成依据。
3. 每个任务尽量单独提交，提交说明使用中文并写清楚影响范围。
4. 不把构建产物、临时日志、无关截图带入提交。
5. 数据库迁移编号冻结：下一轮新增迁移必须确认已有最大编号，避免并行冲突。
6. 后端接口口径以测试、路由和前端调用三方一致为准；不允许只改测试绕过问题。

## 5. Phase 0：发布闸门修复

### Task 0.1 修复管理前端 Settings 构建失败

优先级：P0

负责人角色：管理前端工程师

涉及文件：

- `admin-frontend/src/views/Settings/index.vue`

问题描述：

`contact_qq` 校验器中正则字面量直接位于语句起始位置，Vue SFC 编译报 `Unexpected token`，导致 `admin-frontend npm run build` 失败。

建议修复：

```js
const ok = /^\d{5,15}$/.test(value)
ok ? cb() : cb(new Error('QQ 应为 5-15 位数字'))
```

验收命令：

```bash
cd admin-frontend
npm run build
```

完成标准：

- 构建 exit code 为 0。
- 不出现 Vue SFC 编译错误。
- 不引入新的 Settings 表单交互异常。

建议提交信息：

```text
fix(admin-settings): 修复系统设置页 QQ 校验导致的构建失败

将 contact_qq 校验器中的正则判断改为显式变量，避免 SFC 编译时将正则字面量解析为异常表达式。

验证：admin-frontend npm run build 通过。
```

### Task 0.2 修复阅读偏好 fontSize 整数校验

优先级：P0

负责人角色：用户后端工程师

涉及文件：

- `backend/src/controllers/preferenceController.js`
- `backend/tests/controllers/preferenceController.test.js`

问题描述：

`validateAndNormalize()` 当前使用 `Number.parseInt(body.fontSize, 10)`，导致 `16.5` 被截断为 `16` 并错误通过。测试已经覆盖该场景。

建议修复：

- 使用 `Number(body.fontSize)`。
- 使用 `Number.isInteger(n)` 校验整数。
- 保持 12-24 范围不变。

验收命令：

```bash
cd backend
npm test -- tests/controllers/preferenceController.test.js --runInBand
```

完成标准：

- `fontSize: 16.5` 被拒绝。
- 现有偏好控制器测试通过。
- 错误响应 message 与现有测试预期一致。

建议提交信息：

```text
fix(profile-preferences): 严格校验阅读偏好字号为整数

修复 fontSize 使用 parseInt 导致小数被截断通过的问题，确保 12-24 范围内的整数才允许保存。

验证：backend preferenceController 测试通过。
```

### Task 0.3 统一小说状态接口鉴权口径

优先级：P0

负责人角色：用户后端工程师 + 产品负责人确认

涉及文件：

- `backend/src/routes/novels.js`
- `backend/tests/integration/novelStatus.test.js`
- 可能涉及：`ai-xsread-vue3/src/api/novel.js`

问题描述：

当前 `/api/novels/:id/status` 路由使用 `optionalAuth`，但集成测试期望未登录返回 401。数据库不可达时，该接口走到业务查询并返回 500。

推荐口径：

- 将该接口定义为用户态接口，使用 `authenticate`。
- 未登录直接返回 401。
- 登录后返回用户与小说关系状态。

验收命令：

```bash
cd backend
npm test -- tests/integration/novelStatus.test.js --runInBand
```

完成标准：

- 未登录访问 `/api/novels/:id/status` 返回 401。
- 相关 like / bookshelf middleware-only 测试仍通过。
- 如前端存在游客调用状态接口，前端必须容忍 401，不影响小说详情页基础展示。

建议提交信息：

```text
fix(novel-status): 统一小说状态接口为登录态资源

将小说状态接口改为必须登录访问，避免匿名请求在数据库不可达时进入业务查询并返回 500。

验证：backend novelStatus 集成测试通过。
```
