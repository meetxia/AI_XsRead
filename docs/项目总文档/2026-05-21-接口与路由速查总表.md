## 1. 说明

本文档是当前事实入口，覆盖 4 个子工程的全部前后端路由：

- 用户前端 `ai-xsread-vue3/src/router/index.js`
- 用户后端 `backend/src/routes/index.js` 与 `backend/src/routes/*.js`（16 个路由模块）
- 管理前端 `admin-frontend/src/router/index.js`
- 管理后端 `admin-backend/src/routes/index.js` 与 `admin-backend/src/routes/*.js`（10 个路由模块）

口径优先级：**当前代码 > 本文档 > 5/17 旧版速查 > 历史归档**。如果代码与本文档冲突，请同步更新本文档。

## 6. 业务提醒

### 6.1 兼容路径与主路径

- `/api/auth/refresh`（主）和 `/api/auth/refresh-token`（兼容别名）
- `/api/novels/categories`（主）和 `/api/categories`（兼容别名）

新前端调用应优先用主路径。

### 6.2 admin 前缀强制

管理后端所有业务路径**必须**以 `/admin/` 开头，不要和用户端 `/api/` 混写。健康检查 `/health` 不带前缀。

### 6.3 限流细则

- 全局限流（用户后端）：dev 3000 次/15min，跳过 `/api/health` 与 `/uploads/`
- 登录 / 注册专项限流：10 次/15min（`backend/src/routes/auth.js` 的 `authLimiter`）
- 写操作限流：`writeRateLimiter` 中间件应用于点赞/收藏/评分/书架等写路径

### 6.4 双审计表（管理后端）

- `admin_logs`：`novelController` / `userController` / `commentController` 写入
- `code_admin_logs`：`codeController` / `contactController` / `membershipAdminController` 写入

未来计划用 SQL VIEW `v_admin_audit` 整合，详见 [后台功能 Backlog](2026-05-21-后台功能Backlog.md)。

### 6.5 中间件层级

管理后端在 `routes/index.js` 之前未统一挂 `authMiddleware`，**鉴权由各路由模块自行声明**：
- 大部分子路由文件使用 `router.use(authMiddleware)`（在文件顶部）
- 高权限端点额外加 `adminMiddleware`（如设置 VIP / 创建批次 / 作废激活码 / 上传二维码 / 修改配置）

新增管理端路由时务必在文件顶部声明 `authMiddleware`，不要假设全局守卫已生效。

### 6.6 Swagger 文档

`/api-docs` 与 `/api-docs.json` **仅在 dev 环境暴露**（`backend/src/app.js` 用 `if (config.server.env !== 'production')` 包裹）。生产环境访问会 404。
