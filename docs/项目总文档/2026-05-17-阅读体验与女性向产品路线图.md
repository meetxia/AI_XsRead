# 阅读体验与女性向产品路线图

文档日期：2026-05-17

## 1. 定位

本文吸收 2025-10-27 旧 UX 报告中仍有价值的建议，并结合 2026-05-17 当前项目现状重写。旧 UX 报告不再作为当前优先级来源。

当前产品目标不是“页面更好看”，而是让女性向小说读者愿意长期阅读、收藏、互动和回访。

## 2. 路线原则

1. 先保证真实阅读体验，再做装饰性动效。
2. 先保证真实数据，再做精美运营数字。
3. 先做手机断点，再补桌面增强。
4. 先收敛已有能力，再增加大功能。
5. 体验验收必须落到具体页面和具体交互。

## 3. 阅读页路线

### 当前能力

- 章节加载
- 阅读设置
- 翻页控制
- TTS
- 护眼/夜间
- 段评气泡
- 划线
- 书签
- 阅读进度同步

### 下一步

- 多断点阅读排版验收
- 设置面板不遮挡正文
- TTS 控制条不抢阅读空间
- 段评气泡点击范围清晰
- 划线工具条定位稳定
- 翻页模式切换可预期
- 进度保存失败时有静默重试或提示

## 4. 详情页路线

### 当前能力

- 小说详情
- 作者入口
- 评分/收藏/点赞状态
- 章节列表
- 评论/热评/相似推荐相关组件

### 下一步

- 评分人数和在读人数真实化
- 作者其他作品更清晰
- 同标签推荐可解释
- 评论入口与段评/想法关系明确
- 未登录状态下的收藏/点赞引导更柔和

## 5. 书架与个人中心路线

### 当前能力

- 书架
- 历史
- 个人中心
- 关注作者
- 成就
- 书签、划线、想法聚合页

### 下一步

- 书架分类口径统一
- 阅读历史真实同步
- 个人统计真实化
- 成就触发规则和展示规则明确
- 空状态不冷冰冰，也不伪造数据

## 6. 搜索与发现路线

### 当前能力

- 首页
- 发现
- 搜索
- 热门搜索
- 推荐列表
- 分类与标签

### 下一步

- 搜索建议更快更准
- 搜索历史可管理
- 筛选条件清晰
- 推荐原因可解释
- 榜单和标签不要互相抢入口

## 7. 社区互动路线

### 当前能力

- 评论
- 段评
- 划线
- 想法
- 关注作者

### 下一步

- 段评与整本评论的边界明确
- 点赞/取消点赞反馈一致
- 删除/举报/屏蔽规则补齐
- 热门想法和最新评论排序规则明确
- 管理后台能审核或处理不当内容

## 8. 移动端体验路线

重点断点：

- 360px
- 412px
- 768px
- 1024px
- 1440px

验收重点：

- 底部导航不遮挡
- 阅读页按钮命中区足够
- 弹层高度可滚动
- 表单输入不被键盘遮挡
- 卡片文案不溢出
- 管理端在窄屏至少可读可操作

## 9. 情感化设计路线

可保留方向：

- 温柔的空状态
- 阅读成就反馈
- 连读提醒
- 章节完成反馈
- 收藏/关注后的轻反馈

暂缓方向：

- 过重动画
- 大面积装饰性插画
- 没有数据支撑的运营数字
- 影响阅读注意力的强社交提醒
