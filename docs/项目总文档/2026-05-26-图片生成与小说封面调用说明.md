# 图片生成与小说封面调用说明

本文档给后续智能体使用，说明在本项目中如何调用图片生成能力，尤其是小说封面、普通图片、人物图编辑、背景替换和带中文文字封面的生成。

本文档不是测试报告，不记录历史案例。执行任务时以本文档作为调用口径。

## 1. 可用 Skill

当前与图片生成相关的主要 skill 有两个：

| Skill | 用途 | 适用场景 |
|---|---|---|
| `imagegen` | 通用图片生成与编辑 | 小说封面、人物图、场景图、背景替换、透明图、普通视觉资产 |
| `ai-text-cover-generator` | 基于已有图片生成带文字封面 | 用已有截图、产品图、参考图生成营销封面或图文封面 |

优先级建议：

1. 需要生成全新图片或编辑图片时，优先使用 `imagegen`。
2. 需要基于已有截图/产品图加中文标题、副标题、徽章文字时，可使用 `ai-text-cover-generator`。
3. 小说封面通常使用 `imagegen`，因为小说封面需要完整画面构图、人物、背景、氛围和文字控制。
4. 如果用户明确说“用小说封面 skill”，应按小说封面任务组织 prompt，再用 `imagegen` 或项目现有小说封面封装脚本执行。

## 2. 环境变量

本项目图片生成调用通常从 `backend/.env` 读取 OpenAI 兼容接口配置：

```env
OPENAI_BASE_URL=
OPENAI_API_KEY=
IMAGE_MODEL=
IMAGE_TEXT_MODEL=
IMAGE_SIZE=
IMAGE_QUALITY=
OPENAI_FALLBACK_BASE_URL=
OPENAI_FALLBACK_API_KEY=
OPENAI_FALLBACK_IMAGE_MODEL=
OPENAI_FALLBACK_TEXT_MODEL=
```

注意：

- 不要在日志、文档、提交信息、聊天记录中输出密钥值。
- 执行命令时可以临时把 `backend/.env` 中的值注入当前进程环境。
- `.env` 不进 git，不要把真实密钥写入可提交文件。

PowerShell 中常用注入方式：

```powershell
$env:OPENAI_API_KEY=(Select-String -LiteralPath 'backend\.env' -Pattern '^OPENAI_API_KEY=').Line -replace '^OPENAI_API_KEY=',''
$env:OPENAI_BASE_URL=(Select-String -LiteralPath 'backend\.env' -Pattern '^OPENAI_BASE_URL=').Line -replace '^OPENAI_BASE_URL=',''
```

## 3. 模型选择

当前推荐默认模型：

```env
IMAGE_MODEL=gpt-image-2
```

可调用模型名通常包括：

```text
gpt-image-1
gpt-image-1.5
gpt-image-2
gpt-image-2k
gpt-image-4k
```

调用口径：

- 默认使用 `gpt-image-2`。
- 不要仅因为模型名带 `2k` / `4k` 就认为会自动输出 2K/4K。
- 要控制输出分辨率，必须显式传入 `--size`。
- `auto` 适合快速试验比例，但不适合高清定稿。
- 如果用户要求比较多个模型，应并行调用多个模型，不要顺序等待。

## 4. 分辨率与尺寸

### 4.1 推荐尺寸

小说封面常用竖版比例：

| 用途 | 推荐尺寸 | 说明 |
|---|---:|---|
| 快速试稿 | `1024x1536` | 2:3，速度较快 |
| 常规高清封面 | `1536x2048` | 3:4，清晰度与耗时较平衡 |
| 定稿级封面 | `2160x2880` | 3:4，更高清，耗时更长 |
| 接近原图比例编辑 | `1088x1456` | 接近 `1080x1442`，且满足 16 倍数约束 |

### 4.2 尺寸规则

`gpt-image-2` 对尺寸有约束：

- 宽高必须是 `16` 的倍数。
- 最大边不超过 `3840px`。
- 长边与短边比例不超过 `3:1`。
- 总像素数不能超过模型限制。

常用安全尺寸：

```text
1024x1536
1536x2048
2160x2880
2048x2048
2048x1152
3840x2160
2160x3840
```

注意：

- prompt 中写“3:4”只能影响构图倾向。
- 真正决定输出像素尺寸的是 `--size`。
- 如果不传 `--size`，CLI 默认是 `auto`，实际输出尺寸由接口决定。

图片生成结果建议放在：

```text

```

例如：

```text
output/imagegen/novel-covers/
output/imagegen/character-design/
output/imagegen/background-edit/

```

命名建议：

```text
<asset-slug>-<model>-<size>.png

```

规则：

- 不要覆盖已有结果，除非用户明确要求。
- 对比任务使用语义化文件名，方便后续复查。
- 项目要实际引用的图片，不要只留在 `$CODEX_HOME/generated_images`，应复制或移动到项目目录。

## 6. 使用 `imagegen` 生成全新图片

CLI 路径：

```text
C:\Users\Administrator\.codex\skills\.system\imagegen\scripts\image_gen.py
```

PowerShell 模板：

```powershell
$env:OPENAI_API_KEY=(Select-String -LiteralPath 'backend\.env' -Pattern '^OPENAI_API_KEY=').Line -replace '^OPENAI_API_KEY=',''
$env:OPENAI_BASE_URL=(Select-String -LiteralPath 'backend\.env' -Pattern '^OPENAI_BASE_URL=').Line -replace '^OPENAI_BASE_URL=',''

$prompt=@'
在这里写完整图片生成提示词。
'@

python 'C:\Users\Administrator\.codex\skills\.system\imagegen\scripts\image_gen.py' generate `
  --model gpt-image-2 `
  --prompt $prompt `
  --size 1536x2048 `
  --quality medium `
  --output-format png `
  --out 'output\imagegen\task-name\result.png'
```

常用参数：

| 参数 | 说明 |
|---|---|
| `generate` | 生成全新图片 |
| `--model` | 模型名，默认推荐 `gpt-image-2` |
| `--prompt` | 提示词 |
| `--prompt-file` | 从文件读取提示词 |
| `--size` | 输出尺寸，建议显式传入 |
| `--quality` | `low` / `medium` / `high` / `auto` |
| `--output-format` | `png` / `jpeg` / `webp` |
| `--out` | 输出文件 |
| `--force` | 允许覆盖同名文件，谨慎使用 |

## 7. 使用 `imagegen` 编辑已有图片

适用场景：

- 换背景
- 保留人物但改变环境
- 修改光线、季节、天气
- 局部物体替换
- 多图合成

PowerShell 模板：

```powershell
$env:OPENAI_API_KEY=(Select-String -LiteralPath 'backend\.env' -Pattern '^OPENAI_API_KEY=').Line -replace '^OPENAI_API_KEY=',''
$env:OPENAI_BASE_URL=(Select-String -LiteralPath 'backend\.env' -Pattern '^OPENAI_BASE_URL=').Line -replace '^OPENAI_BASE_URL=',''

$prompt=@'
编辑这张图片。明确说明哪些部分必须保持不变，哪些部分需要改变。
'@

python 'C:\Users\Administrator\.codex\skills\.system\imagegen\scripts\image_gen.py' edit `
  --model gpt-image-2 `
  --image 'C:\path\to\input.jpg' `
  --prompt $prompt `
  --size 1536x2048 `
  --quality medium `
  --output-format png `
  --out 'output\imagegen\task-name\edited.png'
```

编辑 prompt 必须写清楚：

- 要保留什么：脸、身份、姿势、衣服、手、道具、构图、比例。
- 要改变什么：背景、天气、光线、风格、场景。
- 不要改变什么：五官、表情、肤色、身体比例、重要物体。
- 禁止项：水印、文字、logo、多余人物、变形、低清、伪影。

如果用户要求“基于这个人物图”，不要只写“像这个人”，要明确：

```text
Preserve the same person, same face, same expression, same hairstyle, same pose, same clothing, same hand position, and same foreground object.
```

## 8. 小说封面生成 Prompt 结构

小说封面 prompt 建议包含以下模块：

```text
生成一张原创中文小说封面图，竖版构图。

小说名：
《书名》

副标题：
副标题文本

类型：
女性向 / 古风 / 现言 / 悬疑 / 治愈 / 奇幻 / 权谋 / 穿越 / 重生 等。

画面内容：
主角是谁，在哪里，做什么，背景是什么，情绪是什么。

构图：
主标题位置，人物位置，背景层次，是否保留标题空间。

文字要求：
中文标题必须完整准确，不要错字、不要繁体、不要乱码、不要伪中文。
副标题必须完整准确，字体小于主标题。

风格：
真实可上架的中文网文封面质感，精致、商业化、电影感。

限制：
不要英文，不要拼音，不要水印，不要品牌 logo，不要二维码，不要作者名。
人物、建筑和文字不要被裁切。
```

如果中文文字准确性很重要：

- 标题不要太长。
- 副标题尽量短。
- 不要同时要求太多文字。
- 生成后必须人工检查，必要时 OCR 复核。
- 如果文字错了，缩短文案并重新生成。

## 9. 小说封面推荐参数

试稿：

```powershell
--model gpt-image-2 --size 1024x1536 --quality medium
```

常规高清：

```powershell
--model gpt-image-2 --size 1536x2048 --quality medium
```

定稿：

```powershell
--model gpt-image-2 --size 2160x2880 --quality medium
```

是否使用 `high` quality：

- 如果用户要求更高质量，可以尝试 `--quality high`。
- `high` 可能更慢，也可能更贵。
- 批量试稿不建议默认 `high`。

## 10. 使用 `ai-text-cover-generator`

该 skill 更适合“已有图片 + 加文字封面”的场景。

典型用途：

- 微信封面
- 工具介绍封面
- 产品截图封面
- 课程封面
- 基于已有图生成带标题视觉

脚本路径：

```text
C:\Users\Administrator\.codex\skills\ai-text-cover-generator\scripts\generate_text_cover.py
```

PowerShell 模板：

```powershell
python "$env:CODEX_HOME\skills\ai-text-cover-generator\scripts\generate_text_cover.py" `
  --image "path\to\source.png" `
  --out "output\imagegen\text-cover\cover.png" `
  --title "主标题" `
  --subtitle "副标题" `
  --badge "徽章文字" `
  --base-url "$env:OPENAI_BASE_URL"
```

注意：

- 这个 skill 是让模型直接渲染文字，不是本地排版。
- 中文文字仍需人工检查。
- 如果标题错字，优先缩短标题并重新生成。
- 如果重要 UI 被遮挡，要在 prompt 中明确只在空白区域加文字。

## 11. 并行调用规则

当用户要求比较多个模型、多个尺寸、多个版本时，应并行调用。

适合并行：

- 同一 prompt，对比多个模型。
- 同一 prompt，对比多个分辨率。
- 同一输入图，对比多个编辑参数。

不建议串行：

- `foreach` 顺序跑多个模型。
- 等一个模型完成后再跑下一个模型。

在 Codex 中应使用 `multi_tool_use.parallel` 发起多个独立命令。

输出文件必须使用不同文件名，避免互相覆盖：

```text
cover-gpt-image-2-1536x2048.png
cover-gpt-image-2-2160x2880.png
cover-gpt-image-4k-auto.png
```

## 12. 生成后验收

每次生成或编辑完成后，至少检查：

1. 文件是否存在。
2. 图片是否可正常读取。
3. 实际尺寸是否符合传入 `--size`。
4. 画面主体是否符合 prompt。
5. 小说封面文字是否正确。
6. 人物编辑是否保留身份、脸、姿势和重要道具。
7. 是否有水印、logo、伪文字、多余人物、严重变形。

Python 校验模板：

```powershell
python -c "from PIL import Image; p=r'output\imagegen\task-name\result.png'; im=Image.open(p); im.verify(); print('ok')"
```

读取尺寸模板：

```powershell
python -c "from PIL import Image; p=r'output\imagegen\task-name\result.png'; im=Image.open(p); print(im.size)"
```

## 13. 对比图与裁切图

当用户需要比较模型或分辨率时，应生成 contact sheet。

建议输出：

```text
<task-name>-contact-sheet.jpg
<task-name>-scaled-crops-contact-sheet.jpg
```

对比不同分辨率时，不要只裁固定像素区域。应按分辨率比例换算同等画面区域：

```text
基准图宽度：1088
基准裁切：300x300

1536 宽图等效裁切 = 300 * 1536 / 1088 = 424
2160 宽图等效裁切 = 300 * 2160 / 1088 = 596
```

然后统一缩放到同一显示尺寸，便于比较细节密度。

## 14. 透明背景

默认不要直接要求 `gpt-image-2` 输出透明背景。

原因：

- `gpt-image-2` 不支持 `--background transparent`。
- 透明背景通常走 chroma-key 生成，再本地移除背景。

默认流程：

1. 生成纯色绿幕或洋红背景图。
2. 使用 `remove_chroma_key.py` 移除背景。
3. 检查 alpha 通道和边缘。

只有在用户明确要求原生透明，且确认可以使用 fallback 时，才考虑：

```powershell
--model gpt-image-1.5 --background transparent --output-format png
```

## 15. 常见错误

### 15.1 中文标题错误

处理方式：

- 缩短标题。
- 删除复杂副标题。
- 重复强调“必须逐字准确”。
- 重新生成。
- 必要时后期本地排版文字，而不是让模型渲染。

### 15.2 生成尺寸不是预期

原因：

- 没有传 `--size`。
- 使用了 `auto`。
- prompt 里的比例说明不会强制像素尺寸。

处理方式：

- 显式传入 `--size`。
- 确认宽高为 `16` 的倍数。

### 15.3 编辑人物时脸变了

处理方式：

- 在 prompt 中加强身份保持：
  - same person
  - same face
  - same expression
  - same hairstyle
  - same pose
  - same clothing
- 降低不必要的风格化要求。
- 避免要求“重绘成插画”，除非用户明确需要。

### 15.4 背景替换不自然

处理方式：

- 要求 match lighting / color temperature / depth of field。
- 明确 foreground person photorealistic。
- 不要同时要求大幅改变人物衣服或姿势。

## 17. 最终交付口径

完成图片生成任务后，应向用户提供：

1. 输出文件路径。
2. 实际尺寸。
3. 使用的模型。
4. 使用的 `size` 与 `quality`。
5. 如有多张图，提供 contact sheet。
6. 如是小说封面，提醒用户检查中文文字。
7. 如是人物编辑，提醒用户检查身份保持和局部细节。
