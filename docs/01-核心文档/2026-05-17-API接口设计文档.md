# API接口设计文档

最后更新日期：2026-05-17

本文档记录 AI-XsRead 当前代码中已经存在的用户端 API 与管理端 API。它不是旧版接口模板的延续，而是根据 `backend/src/routes`、`admin-backend/src/routes`、前端 `src/api` 封装与启动配置重写的主入口文档。

## 1. 服务边界

| 服务 | 目录 | 默认端口 | 开发代理 | 说明 |
| --- | --- | --- | --- | --- |
| 用户 API | `backend` | `8005` | 用户前端 `/api`、`/uploads` 代理到 `http://localhost:8005` | 面向小说阅读、书架、阅读进度、评论互动、上传、作者关注等能力 |
| 管理 API | `admin-backend` | `8001` | 管理前端 `/api` 代理到 `http://localhost:8001` | 面向后台登录、看板、小说、章节、用户、评论、上传 |
| 用户前端 | `ai-xsread-vue3` | `3008` | - | Vue 3 + Vite + Pinia + TailwindCSS |
| 管理前端 | `admin-frontend` | `3010` | - | Vue 3 + Vite + Element Plus + ECharts |

用户 API 在启动时会测试数据库连接并执行 `backend/database/migrate.js` 中发现的待执行迁移。管理 API 使用独立 Express 5 应用，不自动执行用户端迁移脚本。

## 2. 统一响应格式

用户端接口整体以如下格式为目标：

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1715910000000
}
```

管理端接口由各控制器返回，常见字段包括 `code`、`message`、`data`、分页对象或状态文本。由于管理后台仍处于 0.1 阶段，接口返回格式应以控制器实际实现为准，不应再按旧文档假设所有接口完全一致。

## 3. 认证与安全

### 用户端认证

- 登录、注册、刷新令牌由 `backend/src/routes/auth.js` 提供。
- 认证中间件位于 `backend/src/middlewares/auth.js`，受保护接口要求请求头携带 `Authorization: Bearer <token>`。
- JWT 配置从环境变量读取：`JWT_SECRET`、`JWT_REFRESH_SECRET` 必须存在且长度不少于 32 字符。
- 用户端支持刷新接口 `/api/auth/refresh` 与兼容别名 `/api/auth/refresh-token`。

### 管理端认证

- 管理端所有后台业务接口挂载在 `/api/admin/*`。
- `/api/admin/login` 有独立登录限流：15 分钟内最多 5 次。
- `/api` 下启用通用限流：15 分钟内最多 100 次。
- 管理后端数据库配置要求 `DB_HOST`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`。

### 上传与静态资源

- 用户端上传目录通过 `/uploads` 暴露，并显式设置跨域资源头，方便前端从不同端口加载头像、封面等图片。
- 用户端上传接口挂载在 `/api/upload`，支持头像、封面、图片等分类接口，具体字段以 `backend/src/routes/upload.js` 为准。
- 管理端上传接口为 `POST /api/admin/upload/image`，字段名为 `file`。

## 7. 维护原则

1. 新增接口时优先更新对应 `src/routes`、控制器、前端 `src/api` 封装，再更新本文档。
2. 用户端与管理端同名资源不要混写：用户端是 `/api/...`，管理端是 `/api/admin/...`。
3. 路由表以代码为准；本文档用于协作入口，不替代 Swagger 与控制器实现。
4. 需要新增字段时同步检查 `backend/database/migrations`，避免只改接口不改持久化结构。
