# 数据库设计文档

> **文档日期**：2026-05-21（取代 [2026-05-17 版](2026-05-17-数据库设计文档.md)）
> **适用范围**：AI-XsRead 当前生产数据库口径（基础初始化 + 25 个迁移文件后的最终态）
> **来源**：`backend/database/init_step1.sql` + `backend/database/migrations/*.sql` 实际 SQL

## 1. 数据库概况

### 1.1 基本信息

- 数据库类型：MySQL 8.0
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_unicode_ci`
- 业务库名：`ai_xsread`
- 共享方式：用户后端（`backend`）与管理后端（`admin-backend`）**共享同一库**
- 自动迁移：`backend/database/migrations/migrate.js` 在用户后端启动时执行

### 1.2 表分层

当前数据库结构分四层：

1. **基础初始化表**：由 `init_step1.sql` 创建（12 张）
2. **业务迁移表**：由 25 个迁移文件演进补齐（详见 [数据库与迁移现状说明](../项目总文档/2026-05-21-数据库与迁移现状说明.md)）
3. **运行期索引与触发器**：由 `optimization.sql` / `triggers.sql` / `procedures.sql` 维护
4. **管理端独立表**：由 `202605200001__add_admin_tables.sql` 加入流水线

### 1.3 表数量速览

- 基础表：12
- 阅读互动扩展：7（含 5 张新建 + 2 张原有扩展）
- 个人中心 MVP：2
- 会员与激活码：5
- 管理端：5
- 元数据字典：1
- 迁移自身：1（migrations）

**关键约束**：`type` 是 ENUM，**不**用 VARCHAR；`id` 与 `user_id` 都是 `INT UNSIGNED`（Sprint 1 / T-2.2 修订）。

id                INT UNSIGNED PRIMARY KEY AUTO_INCREMENT
batch_id          INT UNSIGNED NOT NULL          -- FK code_batches.id ON DELETE RESTRICT
code_encrypted    VARBINARY(255) NOT NULL        -- iv12 || tag16 || ciphertext
code_lookup_hash  CHAR(64) NOT NULL UNIQUE       -- SHA-256(明文)
code_preview      CHAR(8) NOT NULL               -- 前 4 位 + 后 4 位
code_type         ENUM('monthly','yearly','permanent') NOT NULL
duration_days     INT NOT NULL
status            ENUM('unused','used','void') NOT NULL DEFAULT 'unused'
used_by           INT UNSIGNED                   -- users.id
used_at           DATETIME
void_at           DATETIME
void_by           INT UNSIGNED                   -- admin_users.id
created_at        DATETIME DEFAULT CURRENT_TIMESTAMP
INDEX idx_activation_codes_batch, idx_activation_codes_status, idx_activation_codes_used_by

## 7. 双审计表说明

当前管理端**两张审计表并存**，写入方不同：

| 表 | 写入方 | 字段差异 |
| --- | --- | --- |
| `admin_logs` | `novelController` / `userController` / `commentController` | 含 `module` / `target_id` / `description` / `user_agent` |
| `code_admin_logs` | `codeController` / `contactController` / `membershipAdminController` | 含 `target` / `detail JSON` |

未来计划：

## 9. 迁移与初始化策略

### 9.1 初始化顺序

1. 执行 `backend/database/init_step1.sql`（基础 12 张表）
2. 启动用户后端 `npm run dev` / `npm start`，自动跑 25 个迁移
3. 视环境需要执行 `optimization.sql` / `triggers.sql` / `procedures.sql`

### 9.2 当前迁移特点

- 所有迁移文件 ENGINE=InnoDB + utf8mb4 + utf8mb4_unicode_ci
- 全部幂等（`IF NOT EXISTS` / `ON DUPLICATE KEY UPDATE` / migrate.js 内置忽略 ER_DUP_*）
- 主键统一为 `INT UNSIGNED`（旧版 BIGINT 已修订）
- 命名规范 `^\d{12}__描述.sql$`（强制）

### 9.3 与 5/17 版的差异

5/21 版本相比 5/17 版本新增表：
- 个人中心：`user_preferences` / `notifications`
- 会员激活码：`code_batches` / `activation_codes` / `code_redemption_logs` / `code_admin_logs` / `system_contact`
- 管理端：`admin_users` / `admin_logs` / `banners` / `announcements` / `statistics_daily`
- 元数据：`achievements`

`users` 表新增字段：`vip_level` / `vip_expires_at` / `vip_status` / `read_streak` / `last_read_date` / `total_read_time` / `total_read_words`
`novels` 表新增：`is_vip`
`comments` 表新增：`reply_count`
`chapters` 表新增：`paragraph_hashes`
`bookshelf` 表新增：`last_seen_chapter_id` / `group_name` / `is_top`
