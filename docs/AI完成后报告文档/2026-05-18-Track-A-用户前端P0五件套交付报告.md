### 3) 修改密码全链路（前后端）
- 后端控制器：`backend/src/controllers/authController.js:303-363`
  - 新增 `changePassword`，从 `req.user.id` 取用户、bcrypt 校验旧密码、`bcrypt.hash(newPassword, 10)` 写回 `users.password`；含基础校验：必填、新密码 ≥6 位、新旧不同；返回统一 `Response.success/error`，错误码 400/401/404/500。
- 后端路由：`backend/src/routes/auth.js:22`
  - `router.post('/change-password', authenticate, authController.changePassword)`，复用与 `/me`、`/logout` 一致的 `authenticate` 中间件。
- 前端 API：`ai-xsread-vue3/src/api/auth.js:50-56`
  - 新增 `changePassword`，POST `/auth/change-password`。
- 前端 store：`ai-xsread-vue3/src/stores/user.js:245-263`
  - 真实调用 API，`response.code === 200` 即视为成功并 return true，否则抛错传给 UI。

### 4) SecurityPage 真实化
- 位置：`ai-xsread-vue3/src/views/SecurityPage.vue`（整文件重写）
  - "修改密码"由 `<a href="#">` 改为按钮，触发自实现的轻量对话框（TailwindCSS + 原生 `<dialog>` 风格，未引入新依赖），输入旧密码 / 新密码 / 确认新密码，前端先做必填、长度、一致性校验，再 `userStore.changePassword`；成功后 1.2s 自动关闭。
  - "绑定手机 / 绑定邮箱 / 登录设备管理 / 注销账号"四项不再保留 `href="#"`，改为按钮，点击后用 `alert("XX 功能即将开放")` 明示，并在右侧加"即将开放"标签。
  - 头部返回按钮、`useSeoMeta` 等其他逻辑保持不动。

### 5) 阅读进度同步全链路
- 位置：`ai-xsread-vue3/src/composables/useReadingProgress.js:6-11`、`:62-103`、`:122-134`
  - 顶部 `import { throttle } from 'lodash-es'`、`import { getReadingProgress, updateReadingProgress } from '@/api/readingProgress'`。
  - `syncProgressToServer` 真实调用 `updateReadingProgress({ novelId, ...data })`，外层用 `throttle(..., 1500, { leading:false, trailing:true })` 包裹得到 `throttledSyncProgress`，避免逐页打后端。
  - `saveProgress` 先写本地 `localStorage`、再触发节流同步；API 失败仅 `console.warn`，本地仍然保留。
  - `fetchProgressFromServer` 真实调用 `getReadingProgress(id)` 并取 `response.data`，失败回退 null（沿用原 `mergeProgress` 兜底）。
  - 书签 `addBookmark` / `removeBookmark`（`:174-214`）保持本地存储并加注释明确"书签云端同步不在 Track A 范围、由后续走 `api/bookmarks.js`"，以避免越界。
