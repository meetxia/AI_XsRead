## 五、P2 工程债清单

- 数据库迁移命名规则不统一：`003/004` 数字前缀不符合 `YYYYMMDDHHMI__xxx.sql` 规则，会被 `migrate.js` 跳过
- `patch_v2.sql` 是手工补丁，未纳入迁移框架
- `admin-backend` 无迁移框架，仅有一次性 init 脚本
- `admin_users` 字段定义两份冲突（`docs/06-数据库脚本/admin_tables.sql` vs `admin-backend/scripts/create-admin-tables.js`）
- comments / user_achievements / categories 多文件重复定义，依赖 ON DUPLICATE KEY UPDATE 兜底
- `auth.js` `/refresh` 与 `/refresh-token` 别名重复
- 后端覆盖率门禁过低（30%），前端无阈值
- 无 APM/监控/日志聚合，仅 PM2 文件日志
- `DEPLOYMENT_COMPLETE.md` 日期过期（2025-11-01）

1. 数据库迁移现状混乱（命名不统一、手工补丁、管理端无迁移框架），新环境初始化存在 schema 不一致风险。

4. 项目根目录有 `-w` 这种异常文件名（87 字节），未在本次报告中处理。
