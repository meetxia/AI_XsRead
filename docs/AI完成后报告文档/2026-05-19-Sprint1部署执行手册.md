## 一、部署前必做（5 分钟）

### 1.1 确认本地一切就绪

```bash
cd e:/momo-ruanjiansheji/AI-XsRead

# 1. 确认依赖完整（已加 node-cron / lru-cache / ioredis）
cd backend && npm install && cd ..

# 2. 本机最终验证
cd backend && node -c src/app.js && cd ..
cd ai-xsread-vue3 && npm run build && cd ..
cd admin-frontend && npm run build && cd ..

# 3. 确认 git 干净（或先 push 上 GitHub）
git status
git push origin main      # 推送 7 个 Sprint 1 commit（如还没推）
```

### 1.2 服务器先备份（**关键**）

SSH 登录服务器（或宝塔终端）：

```bash
cd /www/wwwroot/xs.momofx.cn
mkdir -p backups

# 备份数据库（mysqldump 用 root 或有 dump 权限的账号）
mysqldump --single-transaction --routines -u toefl_user -p ai_xsread \
  > backups/ai_xsread_before_sprint1_$(date +%Y%m%d_%H%M%S).sql

# 备份当前代码 commit 与 .env
git -C /www/wwwroot/xs.momofx.cn log --oneline -1 > backups/before-deploy-commit.txt
[ -f backend/.env ] && cp backend/.env backups/backend.env.backup
[ -f admin-backend/.env ] && cp admin-backend/.env backups/admin-backend.env.backup
```

如果 `mysqldump` 报权限错（toefl_user 没 LOCK TABLES 等权限），先解决再继续——**不要跳过备份**。

## 二、用 deploy.py 一键部署（推荐）

### 2.1 一次部署所有（用户端 + 管理端 + 图片）

```bash
# 在本地（Windows）执行
cd e:/momo-ruanjiansheji/AI-XsRead

python backend/scripts/deploy.py --all-with-admin
```

这会：
1. **本地构建** ai-xsread-vue3 → `dist/`、admin-frontend → `dist/`
2. **rsync 增量上传**（按 checksum 比对，没变的不传）：
   - 用户前端 `dist/` → `/www/wwwroot/xs.momofx.cn/ai-xsread-vue3/dist/`
   - 用户后端代码 → `/www/wwwroot/xs.momofx.cn/backend/`
   - 管理前端 `dist/` → `/www/wwwroot/xs.momofx.cn/admin-frontend/dist/`
   - 管理后端代码 → `/www/wwwroot/xs.momofx.cn/admin-backend/`
   - 优化后图片 → `/www/wwwroot/xs.momofx.cn/backend/uploads/`
3. **远端各自 `npm ci --omit=dev`** 安装生产依赖（含 node-cron）
4. **PM2 reload**：`xsread-backend` + `xsread-admin-backend`
5. **后端启动时自动跑 `runPendingMigrations`**：
   - `202605180000__patch_v2_consolidation.sql`（Sprint 1 新增，幂等）
   - 其他迁移如已应用会跳过
6. **健康检查**：curl `http://127.0.0.1:8005/api/health` + `http://127.0.0.1:8001/api/health`
7. **打印迁移 diff**：列出本次新应用了哪些 migration

整个流程预计 5-10 分钟（首次稍慢，后续因 rsync 增量很快）。

### 2.2 分步部署（出问题更好定位）

```bash
# 分步骤跑，每步看日志
python backend/scripts/deploy.py --backend             # 用户后端（含迁移）
python backend/scripts/deploy.py --frontend            # 用户前端
python backend/scripts/deploy.py --admin-backend       # 管理后端
python backend/scripts/deploy.py --admin-frontend      # 管理前端
python backend/scripts/deploy.py --images              # 图片资源
```

### 2.3 先 dry-run 看一眼会做什么

```bash
python backend/scripts/deploy.py --all-with-admin --dry-run
```

只打印命令、不真正执行，可以先确认目标路径、PM2 进程名、健康检查 URL 都对。

## 四、出问题怎么回滚

### 4.1 后端启动失败（迁移失败 / 模块缺失）

```bash
# 在服务器上
cd /www/wwwroot/xs.momofx.cn

# 看是什么报错
pm2 logs xsread-backend --lines 100 --nostream

# 如果是迁移失败 → 还原数据库 + 回滚代码
pm2 stop xsread-backend xsread-admin-backend

# 还原数据库
mysql -u toefl_user -p ai_xsread < backups/ai_xsread_before_sprint1_<时间戳>.sql

# 回滚代码到部署前 commit
git reset --hard $(awk '{print $1}' backups/before-deploy-commit.txt)

# 重新装依赖（因 package.json 退回了）
cd backend && npm ci --omit=dev && cd ..
cd admin-backend && npm ci --omit=dev && cd ..

# 重启
pm2 start ecosystem.config.js
pm2 save
```

### 4.2 前端白屏

```bash
# 检查 dist 是否就绪
ls /www/wwwroot/xs.momofx.cn/ai-xsread-vue3/dist/ | head

# 检查 nginx 配置 + 重载
nginx -t && nginx -s reload

# 浏览器强制不缓存（无痕 + Ctrl+F5）
```

如果还白屏，看 `/www/wwwlogs/xs.momofx.cn_error.log` 找具体错误。

## 五、管理端访问入口（一次性配置）

如果你之前没配过 admin 入口，**部署完管理端 dist 在服务器上但浏览器访问不到**。两种方案二选一：

### 方案 A：独立子域名（推荐，干净）

```nginx
# 加到 nginx 配置（宝塔面板新建一个站点 admin.momofx.cn）
server {
    listen 80;
    server_name admin.momofx.cn;

    root /www/wwwroot/xs.momofx.cn/admin-frontend/dist;
    index index.html;

    location /api {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

然后 DNS 加一条 `admin.momofx.cn` A 记录指向服务器 IP，宝塔申请 SSL。

### 方案 B：主站子路径 `/admin/`

修改 admin-frontend 的 `vite.config.js` 加 `base: '/admin/'`，重新 build 部署。然后在主站 nginx 加：

```nginx
location /admin/ {
    alias /www/wwwroot/xs.momofx.cn/admin-frontend/dist/;
    try_files $uri $uri/ /admin/index.html;
}

location /admin-api/ {
    proxy_pass http://127.0.0.1:8001/api/;
    proxy_set_header Host $host;
}
```

这种方式需要前端 axios baseURL 也改成 `/admin-api`，相对复杂，**推荐方案 A**。

## 六、deploy.py 的关键命令速查

| 场景 | 命令 |
|------|------|
| 全量首次部署 | `python backend/scripts/deploy.py --all-with-admin` |
| 只更新用户后端 | `python backend/scripts/deploy.py --backend` |
| 只更新管理端 | `python backend/scripts/deploy.py --admin` |

| 监听 dist 自动部署 | `python backend/scripts/deploy.py --watch` |
| 跳过本地构建（dist 已有） | 加 `--skip-frontend-build` |
