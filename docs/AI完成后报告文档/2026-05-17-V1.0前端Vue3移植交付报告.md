## 二、API 集成情况

为了"先视觉 1:1 还原，再渐进接 API"，所有页面都采用「Mock 数据 + 静默 API 兜底」策略：

| 页面 | 已接入 API | 失败兜底 |
|---|---|---|
| HomePage | `getRecommendNovels` | mock 4 本 |
| NovelDetailPage | `getNovelDetail` / `getChapterList` / `getComments` | mock 完整数据 |
| ReadingPage | `getChapterContent`（按 query.chapter） | mock 山有木兮第十二章 |
| BookshelfPage | `bookshelfStore.fetchBookshelf` | mock 9 本 |
| SearchPage | `getHotSearch` / `searchNovels` | 本地热搜数据 |
| HistoryPage | `getReadingHistory` | mock 三组 |
| AuthorPage | `getAuthorInfo` / `getAuthorWorks` | mock 沈砚白 + 3 部作品 |
| LoginPage / RegisterPage | `userStore.login` / `userStore.register` | 错误信息显示 |

**好处**：即使后端没启动，前端也能展示完整设计；后端启动后自动覆盖为真实数据。

## 七、已知问题与限制

1. **章节阅读真实数据**：ReadingPage 通过 `?chapter=` query 拉取内容，但路由仍为 `/reading/:id`。如需切章节，需先实现章节列表抽屉再调通。
2. **登录路由守卫未启用**：要 enforcement，需在 `router/guards.js` 中读取 user store 并 beforeEach 拦截，再在 `router/index.js` 引入。当前所有页面都不强制登录。
3. **AI 生图封面**：`BookCover` 预留 `cover` prop，传入真实 URL 即可生效。后续接 API 时只需在响应中带 `cover` 字段。
4. **旧组件库**（NovelCard、ThemeSelector 等）保留但失去样式（旧主题 token 已不存在）。如需复用，需更新引用。
