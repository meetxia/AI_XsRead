## 7. 已修复的问题

1. 用户域 HTTPS API 串线：`https://xs.momofx.cn/api/health` 一度返回管理后端 `{"status":"ok"}`。根因是线上 `xs.momofx.cn.conf` 缺少用户域 443 server block，HTTPS 被其他 443 站点接走。已同步仓库 Nginx 配置并 reload。

2. 定时任务缺字段：17:00 出现 `Unknown column 'n.hot_score'`。已新增 `202605220003__add_novel_hot_score.sql` 并部署执行。

3. 管理端封面 404：管理域 `/uploads` 一开始没有命中真实 uploads 目录；后续发现 `.jpg` 正则 location 抢先匹配。已改为 `location ^~ /uploads/` + 目录 alias，刷新 `/novels` 后控制台 0 error。

4. 旧存储过程依赖：定时任务不再调用缺失的 `sp_batch_update_views` / `sp_calculate_hot_rank` / `sp_clean_expired_tokens`，改为应用层 SQL 与迁移表配套。

## 8. 已知风险与后续建议

- Nginx 仍提示 `admin.xs.momofx.cn` 存在重复 server block，当前实际生效的是独立的 `/www/server/panel/vhost/nginx/admin.xs.momofx.cn.conf`。这次已在该文件补了 `/uploads`，但后续建议整理掉重复 server block，降低运维歧义。
- Nginx 提示 `listen ... http2` 写法 deprecated，建议后续改为独立 `http2 on;`。
- 用户后端 MySQL2 仍有 `acquireTimeout` / `timeout` invalid configuration warning，未来 mysql2 版本可能升级为错误，建议单独治理数据库连接配置。
- npm 依赖有 deprecated warning，尤其 `multer@1.x`，建议排期升级。
- 管理端当前线上只有 `super_admin` 账号，没有 `editor`。本次收紧管理业务路由不会影响现有账号；如果未来启用 `editor`，必须先设计权限矩阵，避免运营账号被误拦。
- 管理前端构建仍有大 chunk 警告，可后续通过路由级拆包和 Element Plus / ECharts 按需优化。

## 9. 回滚预案

如发现线上严重问题，建议按以下顺序回滚：

1. 先停止服务写入：
   ```bash
   pm2 stop xsread-backend xsread-admin-backend
   ```
2. 恢复代码到部署前 commit 或指定稳定 commit，并重新安装依赖：
   ```bash
   cd /www/wwwroot/xs.momofx.cn
   # 按备份 commit marker 或 Git 稳定提交恢复
   cd backend && npm ci --omit=dev --no-audit --no-fund
   cd ../admin-backend && npm ci --omit=dev --no-audit --no-fund
   ```
3. 如需数据库回滚，使用部署前备份：
   ```bash
   mysql ai_xsread < /www/wwwroot/xs.momofx.cn/backups/ai_xsread_before_20260522_165525.sql
   ```
4. 如需 Nginx 回滚，恢复本次备份文件后执行：
   ```bash
   nginx -t && nginx -s reload
   ```
5. 重启 PM2 并重新验收：
   ```bash
   pm2 start /www/wwwroot/xs.momofx.cn/ecosystem.config.js --only xsread-backend --update-env
   pm2 start /www/wwwroot/xs.momofx.cn/ecosystem.config.js --only xsread-admin-backend --update-env
   ```
