# 小说初始统计数据补齐上线报告

## 完成内容

本次处理了线上和本地小说存在初始统计数据缺失的问题，重点补齐小说的阅读量、点赞数、收藏量、评分和评分人数，避免用户端出现“0.0万人在读”等影响观感的数据。

1. 新增通用补齐脚本 `backend/scripts/backfill-novel-initial-stats.js`。
   - 支持 `--target local --dry-run` 检查本地缺失项。
   - 支持 `--target local` 直连本地数据库补齐。
   - 支持 `--target production --dry-run` 检查线上缺失项。
   - 支持 `--target production --confirm-production` 通过线上管理 API 补齐。
   - 只补缺失字段，已有非零数据不覆盖。

2. 扩展管理端小说创建/更新接口。
   - `POST /api/admin/novels` 支持写入 `likes`、`ratingCount` / `rating_count`。
   - `PUT /api/admin/novels/:id` 支持更新 `likes`、`ratingCount` / `rating_count`。
   - 保留原有 `views`、`collections`、`rating` 更新能力。
