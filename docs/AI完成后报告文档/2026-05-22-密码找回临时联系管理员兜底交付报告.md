# 密码找回临时联系管理员兜底交付报告

## 完成了什么

根据当前腾讯云邮箱 / 邮箱验证码服务尚未开通的实际情况，将用户端“忘记密码 / 重置密码”流程临时降级为“联系管理员处理”：

- `/forgot-password` 页面不再承诺发送重置邮件，提交后仅提示“密码找回暂未开放，请联系管理员 QQ：472990945 处理。”
- `/reset-password` 页面不再展示新密码表单，避免用户误以为邮件 token 重置链路可用。
- 新增默认客服 QQ 迁移，公开客服联系方式默认写入 `472990945`。
- 后端 `POST /api/auth/forgot-password` 不再查询数据库、不再创建 `password_reset_tokens`、不再发送邮件或打印重置链接。
- 后端 `POST /api/auth/reset-password` 直接返回 400 联系管理员文案，不再开启事务或修改密码。

## 已知问题

- 密码找回仍未具备自助能力，忘记密码用户需要通过管理员人工处理。
- 前端“联系管理员”按钮当前复制管理员 QQ，尚未直连客服邮箱 / 微信 / 帮助中心联系方式。
- 构建输出提示 `baseline-browser-mapping` / `Browserslist caniuse-lite` 数据偏旧，这是现有依赖数据提示，本次未处理。

## 后续建议

- 腾讯云企业邮箱或验证码服务开通后，再恢复为“邮箱验证码 + 设置新密码”的正式流程。
- 恢复正式流程时建议保留防枚举响应、限流、验证码有效期、一次性使用、日志脱敏。
- 可考虑让“联系管理员”按钮跳转 `/about/help` 或读取 `GET /api/system/contact` 展示客服信息。
