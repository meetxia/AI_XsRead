## 1. 总体结论

**评分：48 / 100（"功能页面齐全，但构建闸门未恢复，链路上存在多个硬伤"）**

实测 `npm run build` 在主代理本机仍然失败：
```
[vite:vue] [vue/compiler-sfc] Unexpected token (86:9)
admin-frontend/src/views/Settings/index.vue
270 |        validator: (_rule, value, cb) => {
271 |          if (!value) return cb()
272 |          /^\d{5,15}$/.test(value) ? cb() : cb(new Error('QQ 应为 5-15 位数字'))
    |           ^
```

根因 = JavaScript ASI（自动分号插入）误判：上一行 `return cb()` 没有分号，下一行行首是 `/`，babel parser 把 `cb() / ^\d.../` 解析为除法表达式，撞 unexpected token。**这是项目最关键的 P0 阻断点**。其他维度：

- 11 个 view 全部存在，路由注册正确，CRUD 闭环（NovelList/NovelEdit/ChapterList/ChapterEdit/UserList/UserDetail/CommentList）。
- API 层 7 个文件齐全，与管理后端控制器一一对应。
- request.js 401 处理 + Token 注入 + ElMessage 集中错误反馈 ✅
- **Login 页明示"默认账号: admin / 密码: admin123"并 reactive 默认填进表单**（`Login/index.vue:52,72-74`）— 安全洁癖警示。
- 路由守卫只看 `getToken()` 是否存在，**没有校验 token 是否真有效**。
- **没有 401 → token 过期专门处理后跳转**：request.js 触发 router.push('/login') 但 Login 页本身又默认填 admin/admin123，实际上用户被踢出后下次进入是"一键自动登录默认管理员"——**反而破坏了 admin/admin123 的安全提醒意义**。
- 没有 vitest / @vue/test-utils / playwright 测试相关依赖（`package.json` 13 个 deps 全是运行时和构建工具，**0 测试**）。
- ECharts 6 整包打入 bundle（1080KB 原始 / 350KB gzip，brotli 283KB），未做按需引入。

整体可以发布给"内部技术团队"使用，对外发布前必须把 P0 收住。

| `stats.html`（rollup-plugin-visualizer 输出） | 1322 KB | 147 KB | 114 KB |

**报错**：`Unexpected token (86:9)` 在 `src/views/Settings/index.vue` 第 272 行（脚本相对偏移 86）。

报错代码：
```js
contact_qq: [
  {
    validator: (_rule, value, cb) => {
      if (!value) return cb()                              // ← line 271
      /^\d{5,15}$/.test(value) ? cb() : cb(new Error('...')) // ← line 272
    },
    trigger: 'blur'
  }
]
```

**ASI 解析**：babel 把第 271/272 行解析为：
```js
return cb() / ^\d{5,15}$/.test(value) ? cb() : cb(...)
```
即"返回 `cb() ÷ 正则字面量`"，行首 `/` 被误判为除号。

**修复方案（任选其一）**：
```js
// 1. 显式分号
if (!value) { cb(); return; }
const ok = /^\d{5,15}$/.test(value)
ok ? cb() : cb(new Error('QQ 应为 5-15 位数字'))

// 2. 用 if/else 而非三元
if (!value) return cb()
if (/^\d{5,15}$/.test(value)) cb()
else cb(new Error('QQ 应为 5-15 位数字'))
```

同样的代码在 `src/views/Settings/index.vue:368-369` **再出现一次**——只要修了 272 行的，368 行的也得同步修。

### 7.2 构建输出路径异常

实测构建产物路径都是 `dist/E:/momo-ruanjiansheji/AI-XsRead/admin-frontend/js/...`——绝对路径被嵌入 dist 文件名前缀。这是 `vite-plugin-compression` 在 Windows 下的已知现象（rollup-plugin-visualizer 也类似）。
