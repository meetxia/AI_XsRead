## 四、数据流说明

### 4.1 `usePreferencesStore` ↔ `useReadingSettingsStore` 同步策略

服务端 `user_preferences` 是 truth，本地 `readingSettings`（用于阅读器渲染）是缓存。两者字段不完全等价，需要双向映射：

| preferences (Agent C) | readingSettings (composable) | 映射逻辑 |
|---|---|---|
| `fontSize` (12-24) | `fontSize` (12-28) | 直接赋值 |
| `lineHeight` (1.4-2.2) | `lineHeight` (1.4-2.2) | 直接赋值 |
| `theme` 'cream'/'night'/'eye' | `background` 'sepia'/'dark'/'green' | `mapThemeToBackground()` |
| `pageMode` 'scroll'/'page' | `pageFlipMode` 'scroll'/'tap' | `mapPageModeToFlip()`（page → tap） |
| `eyeProtection` bool | （无对应字段） | 仅 preferences 内部使用，预览页 filter 体现 |

调用时机：
- `applyState()` / `update()` 内部触发 `syncToReadingSettings()`
- `syncToReadingSettings()` 调用 `useReadingSettingsStore().save({...})`，由 readingSettings 自身的 `watch` 持久化到 `localStorage['reading-settings']`
- 防御性 try/catch 包裹，确保 readingSettings 不可用时不阻断主流程

### 4.2 偏好更新流程（登录态）

```
User 操作（滑块/按钮）
  └── prefs.update({ fontSize: 18 })
        ├── normalize merge → state 立即变化（UI 实时响应）
        ├── syncToReadingSettings() → 阅读器配置同步
        ├── writeLocal(merged) → localStorage['xs:preferences']
        └── debouncedPush(patch, prevSnapshot)（500ms 防抖）
              ├── 成功：updateMyPreferences(patch) → 用 server data 回写 + 更新 lastSnapshot
              └── 失败：回滚到 prevSnapshot + writeLocal + syncToReadingSettings
```

### 4.3 偏好初始化流程

```
PreferencesPage onMounted
  └── prefs.fetch()
        ├── 未登录 → restore() 从 localStorage 取值（无值则 DEFAULTS）
        └── 已登录 → GET /api/user/preferences
                      ├── 成功 → applyState(res.data) + writeLocal
                      └── 失败 → restore() 降级
```

### 4.4 通知中心数据流

```
NotificationsPage onMounted
  └── refresh()
        ├── 未登录 → store.reset() （清空 list / unreadCount = 0）
        └── 已登录 → Promise.all([fetchList, fetchUnreadCount])

点击单条
  └── store.markRead(id)
        ├── 已读 → 直接 return
        └── 未读 → POST /:id/read
                    └── 成功：list 项 isRead=1 + readAt + unreadCount--

点击「全部已读」
  └── store.markAllRead()
        └── POST /read-all
              └── list 全部 isRead=1 + unreadCount = 0
```

### 4.5 登录态切换处理

两个页面均通过 `watch(() => userStore.isLogin)` 响应：
- 登录 → 重新 `fetch` / `refresh`
- 登出 → store.reset() 清空敏感状态

## 五、已知问题与待联调点

1. **build 未跑通验证**：当前会话无法执行 `npm`/`npx`/`node` 命令（permission denied），实际构建结果需主线程在本地复测。文件已经过逐行复核，预期无编译错误
2. **接口降级**：Agent A 的 preferences/notifications 接口若未上线，前端表现：
   - `getMyPreferences` 失败 → fallback 到 localStorage（restore）
   - `listNotifications` 失败 → list 置空 → 空态展示
   - `getUnreadCount` 失败 → unreadCount 保持 0，铃铛红点不显示
   - 全程 console.warn 标记，不会白屏
3. **暗色 theme 配色**：通知 item 的 `bg-amber-50` / `bg-blue-50` 是 tailwind 内置色，未在 `tailwind.config.js` 中扩展但属于默认 palette，能正常工作
4. **铃铛红点联动**：本次只完成 store 端 `unreadCount`，ProfilePage / 全局 header 的铃铛挂接由 Agent D 负责；本 Agent 已确保 `useNotificationStore().unreadCount` 暴露并实时更新
5. **`pageMode = 'page'` → readingSettings 'tap'**：分页模式有「点击翻页」与「滑动翻页」两种，本设计统一映射为 `tap`，若产品想区分需追加 `swipe` 模式开关（不在本次 MVP 范围）
6. **未登录态预览**：未登录用户的偏好仅本机生效，注销/换设备会丢失。已在页面底部以小字提示
