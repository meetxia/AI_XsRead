## 已知问题

- `/uploads/tx/missing.png` 这种不存在的图片会被 SPA fallback 兜回 `index.html` 而不是返回真 404。视觉上 UI 会显示一张破图（`<img>` 加载到的是 HTML），影响很小，未来可在 `location ^~ /uploads/` 里加 `try_files $uri =404;` 修复
- `vite.config.js` 的 `experimental.renderBuiltUrl` 配合 `window.__assetsPath` 这套机制是早期为了某种 CDN 路径方案加的，现在没人用，但不破坏功能。下次大版本可以直接删掉 `experimental` 那段，回归 Vite 默认行为，并清理 `index.html` 里的 shim
- `ProfilePage.vue` 里的"快捷功能 / 阅读统计"区块在已登录态下使用了 `useUserStats`，目前未对接真实 API，全是 0。后续 sprint 完善
