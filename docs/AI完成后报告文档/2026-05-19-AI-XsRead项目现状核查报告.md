项目已经不是“只有基础用户端”的早期状态，而是进入了多模块并行开发后的中后期状态。用户端阅读、书架、个人中心、会员、段评、划线、关注作者、SEO 等功能已经有较多实现和测试；用户后端具备较完整的 Express API、迁移体系和 Jest 覆盖；管理端也已接入小说、章节、用户、评论、统计、设置、会员码等模块。

1. 管理前端生产构建失败，阻断后台发布。
2. 用户后端测试仍有 2 个失败用例，且其中 1 个是鉴权行为与测试预期不一致。
3. 工作区存在大量未提交个人中心相关改动，已能通过用户端构建，但尚未完成全链路收口与代码入库确认。

当前建议状态判断：

- 用户端：基本可构建、单测通过，功能面较完整，但存在大文件和未提交改动风险。
- 用户后端：测试覆盖较强，但当前测试未全绿，偏好接口和小说状态鉴权需要修复。
- 管理前端：功能模块较多，但当前构建失败，是最高优先级工程阻断。
- 管理后端：功能实现存在，但没有有效测试脚本，质量闸门偏弱。
- 数据库：迁移体系已建立，近期新增个人中心偏好和通知迁移仍处于未提交状态。

失败 1：阅读偏好 `fontSize` 整数校验不严

- 文件：`backend/src/controllers/preferenceController.js`
- 相关代码：`validateAndNormalize()` 中使用 `Number.parseInt(body.fontSize, 10)`。
- 现象：测试期望 `fontSize: 16.5` 被拒绝，但当前会被 parseInt 截断为 16，因此通过校验。
- 失败用例：`backend/tests/controllers/preferenceController.test.js`
- 负责人判断：这是明确 bug，应改为 `Number(body.fontSize)` 后检查 `Number.isInteger(n)`，或先拒绝非整数字符串。

失败 2：小说状态接口匿名访问返回 500，而测试期望 401

- 文件：`backend/src/routes/novels.js`
- 当前 `/novels/:id/status` 使用 `optionalAuth`。
- 测试：`backend/tests/integration/novelStatus.test.js` 期望未登录时返回 401。
- 现象：数据库不可达时，该接口走到业务查询并返回 500。
- 负责人判断：这里需要产品和接口口径确认：
  - 如果“小说状态”是用户态资源，应改为 `authenticate`，未登录直接 401。
  - 如果允许游客获取匿名状态，则测试应调整，并且业务层要避免无 DB 时暴露 500。
  - 从当前测试名看，团队最近口径更偏向“必须登录”。

其他测试输出风险：

- 集成测试提示本机数据库不可达：`Access denied for user 'root'@'localhost'`，相关 DB 用例被跳过。
- MySQL2 警告 `acquireTimeout` 和 `timeout` 是无效连接配置项，未来版本可能变成错误。建议清理配置。

后端大文件风险：

阻断点：

- 文件：`admin-frontend/src/views/Settings/index.vue`
- 位置：`contact_qq` 校验器附近。
- 当前代码：

```js
/^\d{5,15}$/.test(value) ? cb() : cb(new Error('QQ 应为 5-15 位数字'))
```

- Vite / Vue SFC 编译报错：`Unexpected token`。
- 负责人判断：这是 JavaScript 自动分号插入 / 正则字面量解析上下文导致的语法问题。建议显式写成：

```js
const ok = /^\d{5,15}$/.test(value)
ok ? cb() : cb(new Error('QQ 应为 5-15 位数字'))
```

构建体积信号：

- 在失败前已输出部分 gzip / brotli 产物信息：
  - `element-plus` chunk 约 956 KB 原始、gzip 约 289 KB。
  - `echarts` chunk 约 1080 KB 原始、gzip 约 350 KB。
  - CSS 约 342 KB 原始、gzip 约 48 KB。
- 后台系统可接受较大依赖，但如果管理端首屏要求较高，后续应做按页加载与 ECharts 按需拆分。

其他风险：
