# MOMO小说 SEO 优化交付报告

文档日期：2026-05-17

## 完成内容

本次按 SEO 专项对用户端与后端索引入口做了并行优化：

1. 增强用户端页面级 SEO 能力：
   - `useSeoMeta` 支持 `robots`、canonical、OG、Twitter Card、`og:locale`。
   - 支持单个或多个 JSON-LD 结构化数据脚本。
   - 页面切换时清理上一页 JSON-LD，避免结构化数据污染。

2. 补齐公开页面 SEO：
   - 首页输出 `WebSite` + `Organization`。
   - 小说详情页输出 `Book` + `BreadcrumbList`。
   - 推荐页、作者页补齐标题、描述、canonical 和结构化数据。

3. 屏蔽低价值页面索引：
   - 登录、注册、书架、历史、个人中心、会员、搜索、安全页、成就页、关注作者页等补 `noindex,follow`。
   - `robots.txt` 同步屏蔽私有/低价值路径。

4. 优化 sitemap / robots：
   - 前端静态 `sitemap.xml` 移除 `/search`、`/bookshelf`、`/history` 等低价值入口。
   - 后端新增动态 `GET /sitemap.xml` 与 `GET /robots.txt`。
   - 动态 sitemap 在数据库可用时追加小说、作者、分类入口；数据库不可用或查询异常时降级到静态公开入口。
   - 数据库查询增加短超时保护，避免爬虫访问 sitemap 时被数据库连接状态拖住。

5. 修复验证中暴露的问题：
   - 登录/注册页白底 logo 改为动态绑定路径，避免 Vitest 将 `/logo-white.svg` 解析为 `file:///logo-white.svg`。
   - 更新注册跳转测试的 `useRoute` mock 与当前邮箱注册表单结构。

## 已知问题

1. 当前用户端仍是 SPA 运行时 SEO。搜索引擎抓首屏 HTML 时，动态小说详情页的 JSON-LD 仍依赖 JS 执行。
2. 生产环境如果 Nginx 优先返回前端静态文件，`/sitemap.xml` 可能拿到静态兜底版本，而不是后端动态版本。
3. `backend/scripts/.deploy-cache.json` 在工作区原本已有改动，本次没有将其纳入 SEO 交付范围。

## 后续建议

1. 生产 Nginx 将 `/sitemap.xml` 和 `/robots.txt` 明确代理到用户后端，或定时将后端动态 sitemap 写入前端静态目录。
2. 对首页、推荐页、小说详情页、作者页做预渲染或 SSR，提高非 Google 爬虫和首屏 HTML 可见性。
3. 给小说详情页增加更稳定的 slug URL，例如 `/novel/123/title-slug`，同时保留 canonical。
4. 后台内容管理增加 `seo_title`、`seo_description`、`canonical_url`、`is_indexable` 字段。
5. 补充 `og-image.png`，兼容不稳定支持 SVG 的社交平台。
