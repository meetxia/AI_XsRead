# 用户端账号邮箱展示优化报告

## 完成内容

- 在用户端个人中心主卡片展示当前账号邮箱，方便用户进入“我的”页面后直接确认登录账号。
- 在账号安全页面新增“当前邮箱”信息区，明确提示邮箱用于登录与账号识别，当前暂不支持前台修改。
- 在编辑资料页面新增只读“账号邮箱”展示，不使用可编辑输入框，也不参与保存资料请求。
- 新增前端回归测试，覆盖个人中心、账号安全、编辑资料三个位置的邮箱展示与编辑资料页邮箱不可编辑约束。

- `cd ai-xsread-vue3 && npm test -- AccountEmailDisplay.test.js`：通过，3 个测试通过。
- `cd ai-xsread-vue3 && npm test`：通过，26 个测试文件、113 个测试通过。

测试与构建过程中出现 Browserslist / baseline-browser-mapping 数据过期提示，属于依赖数据更新提醒，与本次账号邮箱展示改动无关。

## 已知问题

- 当前前台仍不支持修改邮箱，本次仅做只读展示与文案提示。
- 如果历史登录态 localStorage 中的 `userInfo` 缺少 `email`，页面会显示“未绑定”或不展示主卡邮箱；用户重新登录或刷新用户资料后可读取后端返回的邮箱字段。

## 后续建议

- 如后续开放修改邮箱，应增加邮箱验证、二次确认、旧邮箱/新邮箱通知、冷却时间和后台审计记录。
- 可考虑在登录态恢复后主动调用 `/api/user/profile` 同步最新用户资料，减少旧 localStorage 信息不完整的问题。
