## 6. 已知遗留问题

### 6.1 跨子工程产物（不在本 Sprint 范围）

deploy.py dry-run 时发现 `ai-xsread-vue3/dist/stats.html` 仍会被同步——这是**用户前端**的 vite-plugin-visualizer 产物，本 Sprint 仅修了 admin-frontend 的同名问题。**用户端 Sprint 1 backlog**已记。

### 6.2 npm install 时的依赖审计警告

```
11 vulnerabilities (3 moderate, 8 high)
```

主要是 supertest / superagent / formidable 链上的 transitive 依赖。本 Sprint 只装了 dev 依赖，不影响生产 bundle。建议下一个 Sprint 跑 `npm audit fix` 处理。

### 6.3 admin-backend `ai_xsread` vs `ai_xsread_test`

CI 中 user-backend 用 `ai_xsread_test` 数据库，admin-backend 测试如果跑到真实 DB 也走同名 db。本 Sprint smoke 测试不依赖 DB，所以没问题，但未来加 DB 集成测试时需要协调。

### 6.4 子代理工作期间的小漏

- 子代理 admin-backend 加了 `compression` import 但忘了 `npm install --save compression`（主代理后续补上）
- 子代理改了 `src/config/index.js` 的 DB 双兼容，但漏了 `src/config/database.js` 同样要改（主代理后续补上）
