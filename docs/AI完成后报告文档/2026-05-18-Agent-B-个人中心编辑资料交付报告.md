## 4. 已知问题

1. **构建未跑**：会话沙箱限制 `npm run build`，主线程需补跑一次。基于静态比对结果预期为绿色。
2. **保存接口语义**：`updateUserInfo` 当前仅传 `nickname`，未传 `bio / gender / birthday`，与设计契约 `PUT /api/user/profile` 兼容。如后端要求必填某字段，需在后端 controller 内做可选处理（非本 Agent 范围）。
3. **头像和标签独立调用**：保存按钮只串联 `updateUserInfo` + `saveInterestTags`，头像在选择文件即时上传（与设计 §4.1 一致），不会进保存请求；如运营希望"先选后批量保存"，需要进一步重构。
4. **强制重登的 token 失效保护**：`userStore.logout()` 会同时调 `/auth/logout` 接口；若该接口暂时不可用（503/网络错误），已用 `try/catch` 包裹，本地 token 仍会被清空、跳转仍会执行，符合"密码改了以后旧 token 必须失效"的安全意图。
