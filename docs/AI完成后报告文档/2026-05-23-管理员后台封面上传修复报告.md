# 管理员后台封面上传修复报告

## 完成了什么

修复管理员后台上传小说封面后，用户前端不显示封面并出现 `blob:https://admin... ERR_FILE_NOT_FOUND` 的问题。

根因有两层：

1. 管理前端 `NovelEdit.vue` 在上传成功后把 `URL.createObjectURL(file.raw)` 生成的浏览器临时 `blob:` 地址写入 `form.cover`，导致保存到数据库的不是可长期访问的图片地址。
2. 管理后端上传服务原本把图片写入 `admin-backend/uploads/images`，但生产 Nginx 和用户端静态服务实际暴露的是 `backend/uploads`，即使数据库保存 `/uploads/images/...` 也可能找不到真实文件。

本次修复：

1. 管理前端封面上传显式携带 `Authorization: Bearer <admin_token>`，避免 `el-upload` 绕过项目 axios 拦截器导致上传接口未认证。
2. 管理前端只在上传响应成功且存在 `response.data.url` 时写入 `form.cover`，不再保存 `blob:` 临时地址。
3. 管理前端开发环境增加 `/uploads` 代理到用户后端 `8005`，让本地预览与生产静态目录口径一致。
4. 管理后端上传目录改为共享的 `backend/uploads/images`。
5. 管理后端创建/更新小说时拒绝 `blob:` 等不可持久访问的封面地址，只允许空值、`/uploads/...` 或 `http(s)://...`。
6. 新增前后端回归测试，防止该问题再次出现。

## 已知问题

1. 已经保存到数据库里的历史 `blob:https://admin...` 封面值不会被代码自动恢复，需要在后台重新上传封面，或通过数据库脚本把对应小说的 `cover` 更新为真实 `/uploads/images/...` 地址。
2. 管理后端上传服务现在按项目部署口径写入 `backend/uploads/images`，部署时需确保该目录权限允许 `xsread-admin-backend` 进程写入。

## 后续建议

1. 对线上库执行一次只读排查：

```sql
SELECT id, title, cover FROM novels WHERE cover LIKE 'blob:%';
```

2. 对查出的小说在管理后台重新上传封面并保存。
3. 部署后用无痕浏览器检查用户首页推荐书单、小说详情页、管理端编辑页封面预览是否都能加载 `/uploads/images/...`。
