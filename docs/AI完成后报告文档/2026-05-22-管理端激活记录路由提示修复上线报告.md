# 管理端激活记录路由提示修复上线报告

## 一、上线内容

本次修复管理后台在切换到“激活记录”等标签页/菜单项时，偶发弹出“无效的批次 ID”的问题。

根因是管理端激活码详情路由原本为 `codes/:id`，没有限制 `id` 必须是数字。虽然 `codes/redemptions` 静态路由位于详情路由之前，但在路由切换、缓存组件与异常路径场景下，详情页仍可能拿到非数字参数并触发错误提示。

## 二、涉及文件

- `admin-frontend/src/router/index.js`
  - 将批次详情路由从 `codes/:id` 收紧为 `codes/:id(\\d+)`。
- `admin-frontend/src/views/Codes/BatchDetail.vue`
  - 非法批次 ID 不再弹出错误提示，改为静默返回激活码列表。
  - 下载操作增加非法 ID 保护，避免异常参数继续请求接口。
- `admin-frontend/src/__tests__/router.test.js`
  - 新增路由守护测试，确保批次详情路由只匹配数字 ID。
