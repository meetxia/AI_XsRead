# GPT 图像模型分辨率与编辑测试总结

## 1. 测试目的

本次测试围绕小说封面与人物图像编辑两个实际使用场景，验证以下问题：

1. `gpt-image-1`、`gpt-image-1.5`、`gpt-image-2`、`gpt-image-2k`、`gpt-image-4k` 是否可以通过当前 OpenAI 兼容接口正常调用。
2. `gpt-image-2k`、`gpt-image-4k` 是否会在不传入尺寸参数时自动生成更高分辨率图片。
3. `gpt-image-2` 在显式传入 `size` 参数时，是否可以输出 2K / 更高清尺寸。
4. 图像编辑模式下，基于输入人物图替换草原背景时，是否需要显式传入分辨率参数。
5. 如何更合理地对比不同分辨率输出的细节，尤其是发丝、脸部、手部、背景纹理等区域。

## 2. 调用方式

使用本地 `imagegen` skill 的 CLI/API 路径：

```powershell

python C:\Users\Administrator\.codex\skills\.system\imagegen\scripts\image_gen.py edit ...
```

## 4. Auto 尺寸测试：带中文文字小说封面

### 4.1 测试方式

使用同一条带中文标题的小说封面 prompt：

- 主标题：`《凤栖旧梦》`
- 副标题：`一场被月光唤醒的前世约定`
- prompt 中说明 `3:4` 竖版封面
- 命令中不传 `--size`，让 API 使用默认 `auto`

测试模型：

- `gpt-image-2`
- `gpt-image-2k`
- `gpt-image-4k`

输出路径：

- `output/imagegen/novel-cover-auto-text-compare/fengqi-text-auto-parallel-gpt-image-2.png`
- `output/imagegen/novel-cover-auto-text-compare/fengqi-text-auto-parallel-gpt-image-2k.png`
- `output/imagegen/novel-cover-auto-text-compare/fengqi-text-auto-parallel-gpt-image-4k.png`
- 对比图：`output/imagegen/novel-cover-auto-text-compare/fengqi-text-auto-model-contact-sheet.jpg`

![fengqi auto model compare](../../output/imagegen/novel-cover-auto-text-compare/fengqi-text-auto-model-contact-sheet.jpg)

### 4.2 结果

| 模型 | 是否成功 | 自动输出尺寸 | 比例 |
|---|---:|---:|---:|
| `gpt-image-2` | 成功 | `1086x1448` | `3:4` |
| `gpt-image-2k` | 成功 | `1086x1448` | `3:4` |
| `gpt-image-4k` | 成功 | `1086x1448` | `3:4` |

结论：

- 不传 `size` 时，三个模型都会根据 prompt 中的 `3:4` 生成正确比例。
- 但三个模型的自动输出尺寸完全一致，均为 `1086x1448`。
- 因此，当前接口下 `gpt-image-2k` / `gpt-image-4k` 不会因为模型名而自动生成更高分辨率。
- 如果要获得高清输出，必须显式传入 `size` 参数。

## 6. 女孩人物发丝分辨率测试

### 6.1 测试方式

使用同一个女孩半身人像 prompt，重点要求：

- 长黑色微卷发
- 大量独立发丝、碎发、发梢、高光
- 不要把头发画成整块黑色

使用 `gpt-image-2` 分别生成：

- `1024x1536`
- `1536x2048`
- `2160x2880`

输出路径：

- `output/imagegen/girl-hair-resolution-test/girl-hair-gpt-image-2-1024x1536.png`
- `output/imagegen/girl-hair-resolution-test/girl-hair-gpt-image-2-1536x2048.png`
- `output/imagegen/girl-hair-resolution-test/girl-hair-gpt-image-2-2160x2880.png`
- 全图对比：`output/imagegen/girl-hair-resolution-test/girl-hair-full-contact-sheet.jpg`
- 固定像素裁切对比：`output/imagegen/girl-hair-resolution-test/girl-hair-100pct-crops-contact-sheet.jpg`
- 按分辨率比例换算裁切对比：`output/imagegen/girl-hair-resolution-test/girl-hair-scaled-relative-crops-contact-sheet.jpg`

![girl hair full compare](../../output/imagegen/girl-hair-resolution-test/girl-hair-full-contact-sheet.jpg)

![girl hair scaled crop compare](../../output/imagegen/girl-hair-resolution-test/girl-hair-scaled-relative-crops-contact-sheet.jpg)

### 6.2 结果

| 传入尺寸 | 实际输出尺寸 | 文件大小 |
|---|---:|---:|
| `1024x1536` | `1024x1536` | 约 `2.46 MB` |
| `1536x2048` | `1536x2048` | 约 `4.37 MB` |
| `2160x2880` | `2160x2880` | 约 `7.40 MB` |

### 6.3 裁切对比方法修正

第一次做发丝对比时，使用了固定 `512x512` 像素裁切。这种方法适合观察原始像素局部质感，但不适合判断“同一画面区域在不同分辨率下的信息量”。

后续改为按分辨率比例换算裁切：

| 原图尺寸 | 相对 `1024` 宽比例 | 等效裁切区域 |
|---|---:|---:|
| `1024x1536` | `1.0000x` | `320x320 px` |
| `1536x2048` | `1.5000x` | `480x480 px` |
| `2160x2880` | `2.1094x` | `675x675 px` |

然后统一显示到相同尺寸，用于观察同等画面区域下是否具备更多发丝信息。

结论：

- 判断“真高清”不能只看像素尺寸。
- 更合理的观察方式是：对同一相对位置做按比例换算裁切，再统一显示尺寸进行比较。
- 由于三张图是分别生成，并非同一张图的不同分辨率版本，构图与发型存在随机差异，因此该测试只能作为视觉判断参考，不能作为严格像素级证明。

## 8. 关于“真高清”与“放大”的判断

仅看文件尺寸无法判断是否为真高清。`2160x2880` 只能说明输出文件拥有这么多像素，不代表这些像素一定包含真实新增细节。

更合理的判断方法：

1. 同 prompt 生成低分辨率与高分辨率版本。
2. 对关键区域做按分辨率比例换算的同等画面区域裁切。
3. 统一显示尺寸后观察细节密度。
4. 重点看发丝、眼睛、皮肤纹理、衣料纹理、文字边缘、背景草叶等高频细节。
5. 如需更严格判断，可把低分辨率图用本地超分或 Lanczos 放大到高分辨率，再与模型直出高分辨率图对比。

需要注意：

- 多次独立生成的图片不是同一张图的不同分辨率版本，因此不能做严格像素级对比。
- 真正严谨的判断需要同一随机种子或模型支持从同一 latent 输出多尺寸版本；当前测试链路不具备该条件。
- 本次结论更偏向实际可用性评估，而不是模型内部渲染管线证明。

## 9. 总体结论

### 9.1 模型别名

当前接口中以下模型名均可调用：

- `gpt-image-1`
- `gpt-image-1.5`
- `gpt-image-2`
- `gpt-image-2k`
- `gpt-image-4k`

但从本轮测试看：

- `gpt-image-2k` / `gpt-image-4k` 更像可调用别名或后端档位名。
- 不传 `size` 时，它们不会自动生成 2K/4K 尺寸。
- 是否更有画质优势，需要进一步用同尺寸、同任务做更细的主观质量评估。

### 9.2 分辨率控制

真正控制输出分辨率的是 `size` 参数。

已验证 `gpt-image-2` 可输出：

- `1088x1456`
- `1536x2048`
- `2160x2880`

对于小说封面、人物设定图、背景替换图，建议不要依赖 `auto`，而是显式配置尺寸。

### 9.3 推荐配置

试稿或批量生成：

```env
IMAGE_MODEL=gpt-image-2
IMAGE_SIZE=1024x1536
IMAGE_QUALITY=medium
```

常规高清封面：

```env
IMAGE_MODEL=gpt-image-2
IMAGE_SIZE=1536x2048
IMAGE_QUALITY=medium
```

定稿级封面或重点资产：

```env
IMAGE_MODEL=gpt-image-2
IMAGE_SIZE=2160x2880
IMAGE_QUALITY=medium
```

人物图编辑换背景：

```env
IMAGE_MODEL=gpt-image-2
IMAGE_SIZE=1536x2048
IMAGE_QUALITY=medium
```

如输入图比例接近 `3:4`，建议优先使用：

- `1536x2048`
- `2160x2880`

如输入图比例更接近原图 `1080x1442`，可使用接近原比例且满足 16 倍数约束的尺寸：

- `1088x1456`

## 10. 后续建议

1. 在后端图片生成配置中，将 `IMAGE_SIZE` 作为显式配置项，不依赖模型 auto。
2. 默认试稿用 `1024x1536` 或 `1088x1456`，定稿再升到 `1536x2048` 或 `2160x2880`。
3. 针对小说封面中文文字，建议保留人工复核或 OCR 复核，因为模型虽可生成中文，但仍可能出现错字、伪字、笔画变形。
4. 若后续要做更严谨的高清判定，可以新增一个脚本：
   - 输入低清图与高清图
   - 自动生成按比例裁切对比图
   - 自动生成低清本地放大图
   - 对比模型高清直出与本地放大版本
   - 输出 Markdown 报告
