# 审查报告 04 — 管理后端深度审查

文档日期：2026-05-20
审查对象：`admin-backend/`（Express 5.1 + MySQL2 + JWT + multer 2.0）
审查方式：主代理逐文件阅读 + 静态分析（DB 不可达，未跑测试）

---

## 1. 总体结论

**评分：56 / 100（"功能面可用，但安全面与质量面有明显缺口"）**

管理后端的功能实现是覆盖到位的——10 个控制器、10 条路由、10 张管理表的 CRUD 全闭环、激活码体系（含批次/作废/兑换记录/审计日志）做得相当讲究、客服联系信息和系统配置都接通了。但与用户后端相比，管理后端**质量基线明显落后一档**：

1. **零测试**——`npm test` 仍是占位脚本（`Error: no test specified`，package.json:9），10 个控制器一个单测都没有，与用户后端 42 个 test 文件天差地别。
2. **`/admin/login` 没有 trust proxy** —— `app.js:25-38` 限流配置是合理的（登录 5/15min + 全局 100/15min），但因为没有 `app.set('trust proxy', 1)`，nginx 反代后所有 IP 都是 127.0.0.1，**等价于全站共享一个限流桶**。这意味着只要有一个用户被限流，所有人就都登不进。或反过来：攻击者从一个 IP 暴力破解，只要前 5 次失败被限流，第 6 次起换个 cookie 仍可继续——**限流形同虚设**。
3. **`novelController.getList` 第 59 行 `ORDER BY n.${sort} DESC`** 直接拼接查询参数 sort，**SQL 注入活靶子**。虽然是后台接口需要 JWT，**但管理员账号默认 admin/admin123**，这就是真正的链路 — 默认密码 + SQL 注入 = 攻破整个数据库。
4. **`routes/upload.js` 第 29 行 `if (err instanceof multer.MulterError)` 但文件**没有 import multer**——这是 ReferenceError 现场，任何文件大小超限都会让进程崩溃**（被 errorHandler 兜住输出 500，但 `multer is not defined` 写入日志，吵闹且令人困惑）。
5. **chapter 批量导入接口是占位实现**（`routes/chapters.js:25-37` 注释明说"先做占位"，只校验返回 200，根本不写库）。

整体可用，但发布前必须把 P0 安全风险全收住。

## 4. 安全审计（重点章节）

### 4.1 SQL 注入

#### 高危：`novelController.getList` ORDER BY 注入

**位置**：`admin-backend/src/controllers/novelController.js:11-67`

```js
const { sort = 'last_update_time' } = req.query;
// ...
const [novels] = await db.query(
  `... ORDER BY n.${sort} DESC LIMIT ? OFFSET ?`,
  [...params, parseInt(pageSize), offset]
);
```

`sort` 直接来自 query string，没有任何白名单。攻击 PoC：
```
GET /api/admin/novels?sort=id;%20DROP%20TABLE%20users--
```
虽然 mysql2 默认 `multipleStatements: false`（admin-backend `config/database.js` 待复核，但 mysql2 默认是 false），单语句注入很难直接 DROP TABLE，但 `ORDER BY` 注入仍可：
- 时间盲注 `ORDER BY IF(SUBSTRING(password,1,1)='a', SLEEP(5), 1)`
- 布尔盲注 `ORDER BY (SELECT IF(...))`

**严重性**：High。需要 admin 登录但默认密码 admin/admin123 + 限流失效（见 §4.2）→ 实际利用门槛低。

**修复**：
```js
const ALLOWED_SORTS = ['last_update_time', 'created_at', 'views', 'likes'];
const sortField = ALLOWED_SORTS.includes(sort) ? sort : 'last_update_time';
```

#### 已收敛白名单（安全）

- `dashboardController.getRanking`（line 113-119）：`type` 经过 if/else 收敛到 `views/collections/likes`，再用 `${orderField}` 拼接 — 安全。

#### 其他 SQL 全部参数化

`db.query(SQL, [params])` 风格在 `userController` / `commentController` / `contactController` / `codeController` / `membershipAdminController` / `configController` 全部正确。

### 4.2 鉴权 / 限流

#### `app.set('trust proxy')` 缺失

- `app.js` 全文未调用 `app.set('trust proxy', 1)`。
- 影响：nginx 反代后 `req.ip = '127.0.0.1'`，**所有限流共用一桶**：
  - 全局 100/15min → 站内任意 100 个请求就锁全所有人
  - 登录 5/15min → 5 次失败锁所有人
- 这同时是个 **DOS 风险**和**绕过登录限流的途径**（攻击者只需让一个合法用户先打到 5 次失败，自己就解锁了）。

#### JWT 单密钥

- `admin-backend/src/utils/jwt.js:11-23`：access 和 refresh **共用 `config.jwt.secret`**。
- 影响：refresh token 泄露 = 永久 access。
- 用户后端是双密钥（详见报告 02），管理后端弱化了。

#### 默认管理员密码不强制改

- `admin-backend/scripts/init-admin.js:73-91`：bcrypt round 10 加密 `admin123`，console 提示"请立即修改默认密码"，**没有任何强制机制**。
- `users.password` 类型 VARCHAR(255) ✅
- 入口 `authController.login` 没有 last_password_changed_at 字段，无法判断是否使用过默认密码

#### authMiddleware

- `admin-backend/src/middlewares/auth.js:10-32`：从 Authorization header 取 Bearer token，jwt.verify，挂载 req.user。
- ✅ 干净，不像用户后端那样 console.log 泄漏 token
- `adminMiddleware` 检查 role === 'admin' || 'super_admin'（line 38），`superAdminMiddleware` 仅 super_admin（line 47）

### 4.3 文件上传

#### 路由层 `routes/upload.js` 的 ReferenceError

```js
router.use((err, req, res, next) => {
  if (err instanceof multer.MulterError) { // ❌ multer 未 import
    ...
  }
  return Response.error(res, err.message || '上传失败', 500);
});
```

`upload.js` 顶部只 `const upload = require('../services/uploadService');`，没有 require multer。任何 MulterError 触发会先撞 ReferenceError，被外层 errorHandler 吞掉，但日志会出现 `multer is not defined`。

#### `services/uploadService.js`

- multer.diskStorage（line 12-21）：文件名 `image-${ts}-${rand}${ext}`，**不脱敏原文件名**（line 18 用 `path.extname(file.originalname)` 取扩展名，**ext 没有白名单**）
- ✅ 2MB size 限制（line 37）
- ✅ MIME 白名单 `image/*`（line 26-31）—— 但 prefix 匹配，**`image/svg+xml`** 也会通过 → SVG 内嵌 JS 是 XSS 向量
- 路径直接落 `../../uploads/images/`，没有按 user/year/month 分目录

#### `controllers/contactController.js` 上传

- `ALLOWED_MIME` 严格白名单 jpg/jpeg/png/webp（line 53）✅
- 内存存储 → 校验通过后再写盘 ✅
- 文件名严格构造：`contact-qr-${Date.now()}${ext}`，ext 二次白名单（line 165-173）✅
- **质量明显比 services/uploadService.js 高**

### 4.4 IDOR / 越权

- 写操作普遍走 `adminMiddleware`，对越权防得住。
- `userController.getDetail / updateStatus`：用 `req.params.id` 直接查用户——这是合理的，因为是 admin 接口。
- `membershipAdminController.disableMember`：`Number.isInteger + > 0` 校验 ✅

### 4.5 errorHandler

- `admin-backend/src/middlewares/errorHandler.js`（46 行）：极简 — 只处理 ER_DUP_ENTRY / JsonWebTokenError / TokenExpiredError / ValidationError 四种命名错误。
- ⚠ **第 7 行 `console.error('Error:', err)` 直接打印整个 err 对象**——包含 stack，可能含敏感信息（如 SQL 内容、req.body 引用）
- 比用户后端的 errorHandler（130 行 + ErrorCodes 字典 + AppError）弱得多
