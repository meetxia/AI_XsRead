## 二、5 个新增 API 清单

| Method | Path | 入参 | 返回结构 | 关键 SQL 简述 |
|--------|------|------|----------|---------------|
| GET | `/admin/dashboard/category-distribution` | - | `[{ id, name, value }]` | `categories LEFT JOIN novels ON category_id GROUP BY c.id` 取 `status=1` 分类，按数量降序 |
| GET | `/admin/dashboard/user-activity` | `days=7\|30` | `{ dates:[YYYY-MM-DD], values:[number] }` | `reading_history` 按 `DATE(read_time)` 聚合 `COUNT(DISTINCT user_id)`；后端补齐缺失日期，默认 7 天，最大 365 天 |
| GET | `/admin/dashboard/reading-time-distribution` | - | `{ buckets:[{ label, count }] }` | 优先 `users.total_read_time`（patch_v2 字段），不存在时回退 `reading_history GROUP BY user_id SUM(duration)`；分桶 `<1h / 1-5h / 5-10h / 10-30h / 30h+` |
| GET | `/admin/dashboard/novel-stats` | - | `{ totalNovels, totalChapters, totalWords, newThisMonth, avgRating }` | 5 条 `COUNT/SUM/AVG` 对 `novels`/`chapters`，本月新增按 `created_at >= DATE_FORMAT(CURDATE(),'%Y-%m-01')` |
| GET | `/admin/dashboard/top-novels` | `limit=10`（默认 10，上限 50） | `[{ id, title, author, cover_url, views, read_count }]` | `novels LEFT JOIN reading_history GROUP BY n.id ORDER BY read_count DESC, views DESC` |
