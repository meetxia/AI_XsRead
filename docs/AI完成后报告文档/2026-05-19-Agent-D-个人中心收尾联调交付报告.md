## 4. 已知问题

1. **Out-of-scope 色阶残留**：`SecurityPage.vue`、`FollowingAuthorsPage.vue`、`AchievementsPage.vue`、`HotNotesSection.vue`、`NoteSheet.vue` 仍存在 `text-ink-400` / `text-cinnabar-600` / `text-ink-200` 这类未定义类。Tailwind JIT 不会因为未知类阻断构建，只是这些类不会生成样式（视觉上是文本继承父级颜色）。任务边界明确不动这些文件，且 SecurityPage 是 Agent B 已重构完毕的产物——主线程如果想统一收口可以单独安排一次跨文件清理 PR。
2. **`fetchUnreadCount()` 触发时机**：当前覆盖了 ProfilePage `onMounted` + `watch(isLogin)`。设计文档 §4.4 还提到 App.vue 挂载和进入 `/profile/notifications` 时也要拉一次。前者建议主线程在 `App.vue` 里加一行（C 的 store 已就位，调用即可）；后者 NotificationsPage（C 交付）应已在自身 `onMounted` 调过 `fetchList`，未读数则会被 `markRead` / `markAllRead` 在 store 内部自动维护。
3. **客服微信号 `momo_help` 是占位值**：上线前需要由运营替换为真实账号；同样邮箱 `support@momo-novel.com` 可能也需要替换。
4. **关于页协议正文是占位段落**：明确标注「正式版本将在公开上线前完善」，主线程在正式上线前需要替换为法务最终文本。
5. **未跑测试套件**：项目目录下未发现 Vitest 配置（`package.json` 只提供 `vite build`），未在沙箱内运行单元测试；后端 Jest 测试由 Agent A 自行交付，不在 D 范围。

建议按以下顺序串通走一遍，覆盖所有 D 改动到的链路：

1. **未登录**：直接打开 `/about`、`/about/help` —— 验证两个静态页正常渲染、不报错；锚点点击「用户协议」「隐私政策」能平滑滚动到对应区块；FAQ 折叠面板可展开/收起；联系方式「复制」按钮触发后下方出现「已复制：xxx」提示。
2. **登录**：登录后回到 `/profile`，确认顶部铃铛在 `unread-count > 0` 时显示红点，`= 0` 时不显示（可在 DB 手动 update notifications.is_read 验证）。
3. **关注作者反逻辑回归测试**：账号无关注作者时，`FollowingAuthorList` 不渲染但快捷入口「关注作者」格子常驻；账号已关注作者时，`FollowingAuthorList` 出现且不重复。
4. **登录-登出循环**：登录态切换 store 状态（登录拉未读数 → 登出 reset()），可在 DevTools Pinia 面板观察 `notification.unreadCount` 变化。
