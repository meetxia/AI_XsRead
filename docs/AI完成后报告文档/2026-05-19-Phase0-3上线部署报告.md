## 4. 部署中发现并修复的问题

### 问题：管理后端 PM2 进程 online 但 8001 未监听

现象：

- `pm2 status` 显示 `xsread-admin-backend` 为 `online`。
- `curl http://127.0.0.1:8001/api/health` 失败，提示无法连接。
- `ss -ltnp` 初始只看到 `8005`，未看到 `8001`。

根因：

- 本轮代码为支持 Supertest，将管理后端监听逻辑从 `admin-backend/src/app.js` 拆到 `admin-backend/src/server.js`。
- 服务器 `/www/wwwroot/xs.momofx.cn/ecosystem.config.js` 仍配置 `xsread-admin-backend` 启动 `src/app.js`。
- 因此 PM2 进程加载了 Express app，但没有执行 `listen(8001)`。

修复：

- 将 `ecosystem.config.js` 中 `xsread-admin-backend.script` 从 `src/app.js` 改为 `src/server.js`。
- 同步更新 `deploy-guide.md` 与 `AGENTS.md` 的部署前检查项，补充管理后端 `app.js/server.js` 语法检查。
- 已将修正后的 `ecosystem.config.js` 上传到服务器。
- 执行：

```bash
pm2 delete xsread-admin-backend
pm2 start ecosystem.config.js --only xsread-admin-backend --update-env
pm2 save
```

修复后验证：

- `pm2 describe xsread-admin-backend` 显示 script path 为 `/www/wwwroot/xs.momofx.cn/admin-backend/src/server.js`。
- `ss -ltnp` 显示 `8001` 与 `8005` 均在监听。
- `curl http://127.0.0.1:8001/api/health` 返回 `{"status":"ok",...}`。

## 7. 管理后台域名补充配置

用户已在 DNS 中新增主机记录：

- `admin.xs.momofx.cn`
- A 记录：`175.178.27.89`

已在服务器新增 Nginx 配置：

- `/www/server/panel/vhost/nginx/admin.xs.momofx.cn.conf`

配置内容：

- `root /www/wwwroot/xs.momofx.cn/admin-frontend/dist`
- `/api` 反代到 `http://127.0.0.1:8001`
- SPA fallback：`try_files $uri $uri/ /index.html`

验证结果：

| 验证项 | 结果 |
|---|---|
| `Resolve-DnsName admin.xs.momofx.cn` | 解析到 `175.178.27.89` |
| `curl -I http://admin.xs.momofx.cn/` | HTTP 200，返回管理前端 `index.html` |
| `curl http://admin.xs.momofx.cn/api/health` | 返回 `{"status":"ok",...}` |
| `nginx -t` | 通过 |
| `nginx -s reload` | 已执行 |

证书说明：

- 服务器现有证书 Subject Alternative Name 为 `DNS:*.momofx.cn, DNS:momofx.cn`。
- 该通配符证书只能覆盖一层子域，例如 `admin.momofx.cn`。
- `admin.xs.momofx.cn` 是两级子域，不被 `*.momofx.cn` 覆盖，因此当前 HTTPS 会提示证书主名不匹配。
- 如需 HTTPS，有两个选择：
  - 为 `admin.xs.momofx.cn` 单独签发证书。
  - 改用 `admin.momofx.cn`，复用现有 `*.momofx.cn` 证书。

## 8. 回滚预案

如需回滚本次部署：

1. 停止服务：

```bash
pm2 stop xsread-backend xsread-admin-backend
```

2. 还原数据库：

```bash
mysql -u toefl_user -p ai_xsread < backups/ai_xsread_before_20260519_153744.sql
```

3. 回滚代码到 `backups/before-deploy-commit.txt` 记录的版本，并重装依赖：

```bash
git reset --hard $(awk '{print $1}' backups/before-deploy-commit.txt)
cd backend && npm ci --omit=dev && cd ..
cd admin-backend && npm ci --omit=dev && cd ..
```

4. 重启 PM2：

```bash
pm2 start ecosystem.config.js
pm2 save
```

注意：回滚属于高风险操作，执行前必须再次确认备份文件可读、目标 commit 正确。
