## 系统要求

### 硬件要求

**最低配置**:
- CPU: 2核
- 内存: 4GB
- 硬盘: 50GB

**推荐配置**:
- CPU: 4核+
- 内存: 8GB+
- 硬盘: 100GB+ SSD

### 软件要求

- **操作系统**: Linux (Ubuntu 20.04+ / CentOS 7+) / Windows Server 2019+
- **Node.js**: 18.x 或更高版本
- **MySQL**: 8.0 或更高版本
- **Nginx**: 1.18 或更高版本 (可选)
- **Redis**: 6.0 或更高版本 (可选，用于缓存)

## 数据库部署

### 1. 创建数据库

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE ai_xsread CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 创建用户并授权
CREATE USER 'xsread_user'@'localhost' IDENTIFIED BY '您的密码';
GRANT ALL PRIVILEGES ON ai_xsread.* TO 'xsread_user'@'localhost';
FLUSH PRIVILEGES;

# 退出
EXIT;
```

### 2. 导入数据库结构

cd /path/to/AI_XsRead

## 后端部署

### 1. 克隆代码

```bash
cd /var/www
git clone https://github.com/your-org/AI_XsRead.git
cd AI_XsRead/backend
```

### 2. 安装依赖

```bash
npm install --production
```

### 3. 配置环境变量

创建 `.env` 文件：

```bash
cp .env.example .env
nano .env
```

编辑配置：

```env
# 环境
NODE_ENV=production

# 服务器配置
PORT=8005
HOST=0.0.0.0

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=xsread_user
DB_PASSWORD=您的数据库密码
DB_DATABASE=ai_xsread
DB_CONNECTION_LIMIT=20

# JWT 配置
JWT_SECRET=您的超长随机密钥（建议64位以上）
JWT_EXPIRES_IN=24h
JWT_REFRESH_EXPIRES_IN=7d

# CORS 配置
CORS_ORIGIN=https://your-domain.com

# 文件上传
UPLOAD_DIR=./uploads
MAX_FILE_SIZE=10485760

# Redis 配置（可选）
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# 日志配置
LOG_LEVEL=info
LOG_DIR=./logs
```

### 4. 测试运行

```bash
# 测试启动
npm start

# 如果成功，按 Ctrl+C 停止
```

### 5. 使用 PM2 部署

创建 PM2 配置文件 `ecosystem.config.js`：

```javascript
module.exports = {
  apps: [{
    name: 'ai-xsread-backend',
    script: './src/app.js',
    instances: 'max',  // 使用所有CPU核心
    exec_mode: 'cluster',  // 集群模式
    env: {
      NODE_ENV: 'production',
      PORT: 8005
    },
    error_file: './logs/pm2-error.log',
    out_file: './logs/pm2-out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
    merge_logs: true,
    autorestart: true,
    max_restarts: 10,
    min_uptime: '10s',
    max_memory_restart: '500M'
  }]
};
```

启动服务：

```bash
# 启动
pm2 start ecosystem.config.js

# 查看状态
pm2 status

# 查看日志
pm2 logs ai-xsread-backend

# 设置开机自启
pm2 startup
pm2 save
```

## Nginx 配置

### 1. 创建 Nginx 配置文件

```bash
sudo nano /etc/nginx/sites-available/aixsread.conf
```

### 2. 配置内容

```nginx
# 前端 - 用户端
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    # HTTPS 配置（推荐）
    # listen 443 ssl http2;
    # ssl_certificate /path/to/ssl/cert.pem;
    # ssl_certificate_key /path/to/ssl/key.pem;

    root /var/www/AI_XsRead/ai-xsread-vue3/dist;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css text/xml text/javascript application/json application/javascript application/xml+rss application/rss+xml font/truetype font/opentype application/vnd.ms-fontobject image/svg+xml;

    # 静态资源缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # API 反向代理
    location /api {
        proxy_pass http://localhost:8005;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 单页应用路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
}

# 管理后台
server {
    listen 80;
    server_name admin.your-domain.com;

    root /var/www/AI_XsRead/admin-frontend/dist;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # 静态资源缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 1y;
        add_header Cache-Control "public";
    }

    # API 反向代理（管理后台）
    location /api {
        proxy_pass http://localhost:8001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }

    # 单页应用路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 3. 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/aixsread.conf /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl reload nginx
```

## Docker 部署

### 1. 后端 Dockerfile

创建 `backend/Dockerfile`：

```dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY . .

EXPOSE 8005

CMD ["node", "src/app.js"]
```

### 2. 前端 Dockerfile

创建 `ai-xsread-vue3/Dockerfile`：

```dockerfile
FROM node:18-alpine as builder

WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
RUN npm run build

FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 3. Docker Compose

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: aixsread-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: ai_xsread
      MYSQL_USER: xsread_user
      MYSQL_PASSWORD: user_password
    volumes:
      - mysql_data:/var/lib/mysql
      - ./docx/database_init.sql:/docker-entrypoint-initdb.d/01-init.sql
    ports:
      - "3306:3306"

  redis:
    image: redis:alpine
    container_name: aixsread-redis
    restart: always
    ports:
      - "6379:6379"

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: aixsread-backend
    restart: always
    environment:
      NODE_ENV: production
      DB_HOST: mysql
      DB_USER: xsread_user
      DB_PASSWORD: user_password
      DB_DATABASE: ai_xsread
      REDIS_HOST: redis
    ports:
      - "8005:8005"
    depends_on:
      - mysql
      - redis
    volumes:
      - ./backend/uploads:/app/uploads
      - ./backend/logs:/app/logs

  frontend:
    build:
      context: ./ai-xsread-vue3
      dockerfile: Dockerfile
    container_name: aixsread-frontend
    restart: always
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mysql_data:
```

### 4. 启动 Docker 服务

```bash
# 构建并启动
docker-compose up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f backend

# 停止
docker-compose down

# 重启
docker-compose restart
```

## 常见问题

### 1. 数据库连接失败

**问题**: `ECONNREFUSED` 或 `Access denied`

**解决方案**:
```bash
# 检查 MySQL 是否运行
sudo systemctl status mysql

# 检查防火墙
sudo ufw status

# 重置 MySQL 密码
sudo mysql
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'new_password';
FLUSH PRIVILEGES;
```

### 2. 端口被占用

**问题**: `Port 8005 is already in use`

**解决方案**:
```bash
# 查找占用端口的进程
lsof -i :8005

# 杀死进程
kill -9 [PID]
```

### 3. 前端无法访问后端 API

**问题**: CORS 错误或 502 Bad Gateway

**解决方案**:
- 检查后端服务是否正常运行
- 检查 Nginx 配置中的 `proxy_pass` 地址
- 检查后端 CORS 配置

### 4. PM2 进程崩溃

**问题**: 应用不断重启

**解决方案**:
```bash
# 查看错误日志
pm2 logs ai-xsread-backend --err

# 检查内存使用
pm2 describe ai-xsread-backend

# 增加内存限制
pm2 delete ai-xsread-backend
pm2 start ecosystem.config.js --max-memory-restart 1G
```

### 5. 静态资源 404

**问题**: 图片、CSS、JS 无法加载

**解决方案**:
- 检查 Nginx 配置中的 `root` 路径
- 检查文件权限：`sudo chmod -R 755 /var/www/AI_XsRead`
- 检查文件是否存在

## 备份与恢复

### 数据库备份

```bash
# 创建备份脚本
nano ~/backup-db.sh
```

```bash
#!/bin/bash
BACKUP_DIR="/var/backups/mysql"
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR
mysqldump -u xsread_user -p ai_xsread > $BACKUP_DIR/backup_$DATE.sql
gzip $BACKUP_DIR/backup_$DATE.sql

# 保留最近30天的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete
```

```bash
# 添加执行权限
chmod +x ~/backup-db.sh

# 添加到 crontab（每天凌晨2点备份）
crontab -e
0 2 * * * /root/backup-db.sh
```

### 恢复数据库

```bash
# 解压备份文件
gunzip backup_20251028_020000.sql.gz

# 恢复数据库
mysql -u xsread_user -p ai_xsread < backup_20251028_020000.sql
```

## 安全建议

1. **使用 HTTPS**: 配置 SSL 证书（Let's Encrypt）
2. **定期更新**: 及时更新系统和依赖包
3. **防火墙**: 只开放必要端口
4. **强密码**: 使用复杂的数据库和 JWT 密钥
5. **备份**: 定期备份数据库和文件
6. **监控**: 设置日志监控和告警
7. **限流**: 使用 Nginx 限流防止 DDoS
