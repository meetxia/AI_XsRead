# 修复：阅读页变量重复声明错误

## 问题描述
点击首页的小说卡片进入阅读页时出现编译错误：
```
[plugin:vite:vue] [vue/compiler-sfc] Identifier 'touchStartX' has already been declared.
```

## 错误原因
在 `ReadingPage.vue` 文件中，`touchStartX`、`touchStartY`、`touchStartTime` 这三个变量被重复声明了两次：

1. **第 397-399 行**：使用 `ref()` 声明（用于滑动手势）
2. **第 1459-1461 行**：使用 `let` 重复声明（用于翻页模式）

## 修复方案
删除第 1459-1461 行的重复声明，统一使用第 397-399 行已定义的 `ref` 变量，并在后续使用时添加 `.value` 访问器。

### 修改内容

#### 1. 删除重复声明（1459-1461 行）
```javascript
// 删除前
let touchStartX = 0
let touchStartY = 0
let touchStartTime = 0

// 删除后（直接移除这三行）
```

#### 2. 更新 touchStart 函数（1477-1482 行）
```javascript
// 修改前
touchStartX = e.touches[0].clientX
touchStartY = e.touches[0].clientY
touchStartTime = Date.now()

// 修改后
touchStartX.value = e.touches[0].clientX
touchStartY.value = e.touches[0].clientY
touchStartTime.value = Date.now()
```

#### 3. 更新 touchEnd 函数（1496-1499 行）
```javascript
// 修改前
const touchDuration = Date.now() - touchStartTime
const dx = touchEndX - touchStartX
const dy = touchEndY - touchStartY

// 修改后
const touchDuration = Date.now() - touchStartTime.value
const dx = touchEndX - touchStartX.value
const dy = touchEndY - touchStartY.value
```

## 修复结果
✅ 编译错误已解决，无 linter 错误
✅ 阅读页可以正常访问
✅ 触摸手势功能正常工作
